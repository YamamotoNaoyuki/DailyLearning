# 通奏低音のリアルタイム実現——数字付き低音を和音へ写像する脳と手、即興の手続き的自動化
**日付**: 2026-06-05  
**分野**: piano  
**タグ**: #通奏低音 #figuredBass #即興 #ワーキングメモリ #運動制御 #partimento #ruleOfTheOctave

> **この記録の新規性**: 通奏低音（Generalbass / basso continuo）の「実時間実現（realization）」を、和声理論ではなく**鍵盤奏者の運動制御・認知負荷・手続き的自動化**の観点から扱うのは初めて。視奏（[[視奏の認知科学とサッカード制御]]／[[初見演奏の科学_eye-hand_span・知覚的チャンキング・校正者の誤り・ワーキングメモリと予測]]）の「双対」としての即興という軸も新規。／ **関連既習**: [[視奏の認知科学とサッカード制御]]、[[ヴォイシングと声部バランス_旋律の浮き立たせ・内声を歌わせる・和音内の打鍵コントロール]]、[[バッハフーガの演奏技法_多声部の独立性とウォーキングベース]]、[[スロープラクティスの神経科学]]、[[ジャズのコンピングとルートレス・ヴォイシング・ウォーキングベース]]（左手の即興運動最適化）

## 学んだこと

### 1. 問題設定：通奏低音は「リアルタイム制約下の和声コンパイラ」である
通奏低音奏者が手にする入力は、低音線の音符列と、その上下に付された数字・臨時記号だけである。たとえばバス音 G の上に「6」とあれば「Gの上に6度（と3度）を載せた第一転回和音」を指す（数字は**バス音からの音程**を表すスパース符号で、ある音程は文脈から省略される——3度・5度はデフォルトとして数字に現れない）。奏者はこれを**演奏の瞬間に**4声体（ないしそれ以上）の具体的な鍵盤配置へ展開しなければならない。

これは計算機科学の語彙で言えば、**ソース（数字付き低音）→ターゲット（鍵盤上の物理的指配置）への実時間コンパイル**である。しかも制約が重い：
- **スループット制約**：テンポが与える締切（deadline）。次のバス音が来る前に右手のヴォイシングを決定・実行し終えねばならない。
- **状態依存性**：各和音は孤立して最適化できない。前の和音の声部配置（voice leading）が次の選択肢を制約する。**平行5度・平行8度（parallel fifths/octaves）の禁則**は、本質的に「直前状態 → 現状態」の遷移に対する**ハード制約**であり、各和音をローカルに正しく作っても、遷移が違反しうる。
- **観測可能性の制約**：通奏低音は他のパート（旋律楽器・歌）と協調する。C.P.E. バッハ『正しい鍵盤楽器奏法試論』(*Versuch*, 1753/62) は、奏者が独奏旋律の臨時記号や進行を**聴きながら**実現を調整すべきこと、さらに（連続低音が他楽器に重複されず曲想が許せば）**バス線自体を内声を滑らかにするために即興で修正してよい**と説く。つまり入力すら可変である。

### 2. 認知負荷：なぜ「素朴な実現」はワーキングメモリを溢れさせるか
素朴な実現アルゴリズムを考えると負荷の所在が見える：
1. バス音を読む（視覚 → 音名）
2. 数字を音程としてデコードし、絶対音高集合に変換（G + 「6」→ {B, E} などをバスからの加算で計算）
3. その音高集合を4声体に配分（doubling/spacing の決定）
4. 前の和音との声部進行をチェックし、平行5/8度を回避する経路を選ぶ
5. 選んだ配置を運指へ写像し、打鍵する

ステップ2〜4を**1音ごとに逐次的・宣言的（declarative）に**実行すると、ワーキングメモリ（中央実行系）が即座に飽和する。[[初見演奏の科学_eye-hand_span・知覚的チャンキング・校正者の誤り・ワーキングメモリと予測]]で見たeye-hand spanや知覚的チャンキングの議論がそのまま効く：熟達奏者は**「バス音 → 和音」を1音単位で計算しない**。代わりに**バス進行のパターン（複数音のまとまり）を一つのチャンクとして既知の和声スキーマへ照合**する。これが負荷を桁で下げる。

### 3. 自動化の核：Rule of the Octave（オクターブの規則）という「ルックアップテーブル」
パターン照合の中核装置が **Rule of the Octave（regola dell'ottava、règle de l'octave）** である。これは長調・短調の音階の**各度数（scale degree）に、文脈（上行か下行か、前後の和音）に応じた定型和音を一意に割り当てる**規則表だ。歴史的には Bruschi (1711) に近い記述があり、Campion (1716) が命名、18世紀に Fenaroli の partimento 教程で標準形が確立した。

CS的に言えば Rule of the Octave は**「音階度数 → 和音」の事前計算済みルックアップテーブル**である。これを**長期記憶に焼き込む**ことで、実時間の処理は「バス音 → 度数の同定 → テーブル参照 → 配置」へと短絡し、ステップ2〜4の宣言的計算が**手続き的記憶（procedural memory）からの想起**に置き換わる。数字が省略されていても（unfigured bass / partimento）和音が分かるのは、テーブルがデフォルト値を埋めるからだ。これは通奏低音と**partimento（多くは数字なしの低音課題）**を地続きにする。Gjerdingen の **schemata** 理論はこれを拡張し、外声（バスと最上声）のスケール度数対のパターンを「Prinner」「Romanesca」「Fonte」等の名付けられたチャンクとして記述する——奏者の頭の中の**和声マクロ命令セット**である。

### 4. 平行5/8度回避を「運動として」自動化する
平行5/8度の禁則は、宣言的には「直前和音と現和音で、2声が5度（8度）の平行関係になってはならない」という遷移制約だが、実演ではこれをいちいち検査する余裕はない。歴史的教程の解は**禁則をそもそも生成しない手の動きを身につける**ことである：
- **最近接ヴォイスリーディング（共通音保持・反進行）**を右手のデフォルト運動にする。共通音を同じ指で保持し、他声を最寄りの音へ動かせば、平行は構造的にほぼ生じない。これは**制約充足を探索ではなくバイアスとして埋め込む**——脳が違反解を作りにくい運動方策（motor policy）を獲得する。
- バスが大きく跳ぶときは右手を**反進行（contrary motion）**で動かす、という対跳躍の運動ルールも同様にチャンク化される。

Heinichen の *Der General-Bass in der Composition* (1728) や Gasparini の *L'armonico pratico al cimbalo* (1708) はこうした「手の規則」を体系化し、初学者を「raw material（音程・和音構成）」から「refinements（声部の洗練、装飾、模倣）」へ段階的に進ませる。これは [[スロープラクティスの神経科学]]の自動化曲線——前頭前野・宣言的処理から、基底核・小脳が担う手続き的処理への移行——を、18世紀が経験的に設計したカリキュラムとして読める。

### 5. 即興時の脳：制御を「下げる」ことの神経科学
完全な実現は単なる規則適用を超え、装飾・模倣・即興的対旋律を含む。ここで即興の認知神経科学が接続する。Limb & Braun (2008, PLoS ONE) のジャズ即興の fMRI 研究は、即興時に**背外側前頭前野（DLPFC）と外側眼窩前頭皮質の広汎な脱活性化**と、**内側前頭前野（前頭極）の限局的活性化**、さらに**感覚運動野の広範な活性化**を見出した。解釈は、即興が「自己監視・意識的随意制御を担う中枢の抑制を解いて、刺激非依存・内発的な行動を流す」状態だということ。

これは通奏低音奏者の経験則と符合する：**実現を「考えながら」やると破綻する**。負荷理論（dual-process）では、低熟達者は熟慮的な Type-2 処理に依存し、高熟達者は要素を自動化して Type-1 に委ねる。Rule of the Octave とスキーマの長期記憶化は、まさに通奏低音を Type-2 から Type-1 へ移す手段であり、その完成形で初めて DLPFC を「下げて」即興の自由度（装飾・声部の追加）を載せる余裕が生まれる。**自動化が制御の解放の前提条件**になっている。

## 気づき・洞察

- **視奏との「双対性」が鮮明**：視奏は「完全に指定された楽譜 → 運動」への**復号（decode）**、通奏低音は「過少指定された記号 → 運動」への**生成を伴う復号（decode + generate）**である。両者は同じ眼手スパン・チャンキング・予測機構を使うが、通奏低音では**欠損情報を長期記憶のテーブル（Rule of the Octave / schemata）が埋める**点で、より「予測符号化（predictive coding）」的だ。視奏で熟達者が「次に来る音を予測しながら読む」のと、通奏低音で「度数からデフォルト和音を補完する」のは、**同じ生成モデルの2つの使い方**に見える。

- **スパース符号としての数字付き低音は、優れた帯域圧縮**である。3度・5度を省略し、デフォルトから逸脱するときだけ数字を書く——これは**エントロピー符号化**そのもの。よく現れる和音（三和音）に短い符号（数字なし）、稀な和音（属7・減七・係留）に長い符号（複数の数字）を割り当てている。記譜の節約は同時に、奏者の**予測モデル（何がデフォルトか）への依存を制度化**している。書かれていないものを補えることが、奏者であることの定義になっている。

- **禁則の「運動への内在化」は、制約最適化を探索からヒューリスティックへ落とす工学的判断**だ。平行5/8度回避を毎回探索するのは組合せ的に高価だが、「共通音保持＋最近接進行」というデフォルト運動方策を持てば、制約を満たす解が**ほぼ自動で生成される**。最適ではないが、締切内に「十分良い」解を出す——リアルタイムシステムの設計思想と同型。

- **C.P.E. バッハの「バス線すら即興で直してよい」は深い**。入力が固定でないということは、奏者がコンパイラであると同時に**ソースの編集者**でもあるということ。これは譜面を不可侵とみなす後世のウルテクスト的態度（[[ウルテクスト版と校訂版の楽譜学]]）とは正反対の演奏文化で、「楽譜＝完全な仕様」ではなく「楽譜＝協調的生成の足場」という認識を示す。

## 他分野との接続

- **CS / コンパイラ・リアルタイムシステム**：数字付き低音 → 鍵盤配置は JIT コンパイル、Rule of the Octave はルックアップテーブル／メモ化、禁則回避はデフォルト運動方策によるヒューリスティック制約充足、テンポは締切。スパース記譜はエントロピー符号化。即興の「DLPFC を下げる」は、最適化器を切って学習済み方策をそのまま流すインファレンス・モードに相当。
- **music分野（鑑賞・和声理論）**：Rule of the Octave・partimento・Gjerdingen schemata は和声分析の語彙でもあるが、本記録は**それを「弾く脳の中のデータ構造」として**扱う点で鑑賞分析と棲み分ける。バロック様式の「あの響き」が、奏者の長期記憶テーブルの帰結であるという視点。
- **解剖学・運動制御（[[スロープラクティスの神経科学]]）**：宣言的（前頭前野）→ 手続き的（基底核・小脳）への自動化曲線。通奏低音教程はこの移行を経験的に設計した最古級のカリキュラムの一つ。
- **ジャズ（[[ジャズのコンピングとルートレス・ヴォイシング・ウォーキングベース]]）**：コード記号（C△7, Dm7…）からヴォイシングを即興する現代ジャズは、機能的に**通奏低音の直系**。ルートレス・ヴォイシングや「ガイドトーン優先」は、最近接進行で禁則を避ける18世紀の手の規則と運動論的に同型。即興の fMRI 研究がジャズで得られたのも偶然ではない。

## 次に深掘りしたいこと

- **partimento 課題の運動学習設計**：Fenaroli/Durante の partimento を現代に再導入したとき、Rule of the Octave のテーブル化に要する練習量・転移（[[ハノン・チェルニー等メカニカル練習曲の功罪と運動学習科学_特異性・転移・フォーカルジストニア]]の特異性問題）はどう測れるか。スキーマ単位の interleaving は有効か。
- **chord priming の反応時間実験**：度数 → 和音の想起が本当に「ルックアップ」なら、スキーマ内のチャンクは反応時間で識別可能なはず。和音同定の RT を指標に、熟達者の「テーブル化」を定量できないか。
- **predictive coding としての通奏低音**：欠損情報の補完を、視奏の予測機構（[[初見演奏の科学_eye-hand_span・知覚的チャンキング・校正者の誤り・ワーキングメモリと予測]]）と同じベイズ生成モデルで統一的に記述できるか。視奏＝復号、通奏低音＝生成つき復号、という双対の定式化。
- **チェンバロ／オルガンの発音機構と実現の関係**（frontier 既掲）：エスケープメントのないチェンバロ・オルガンでは打鍵速度が音量に効かないため、ヴォイシングの「重み付け」が**音価・アルペジエーション・声部数の操作**へ移る。発音機構が実現戦略をどう規定するか。
