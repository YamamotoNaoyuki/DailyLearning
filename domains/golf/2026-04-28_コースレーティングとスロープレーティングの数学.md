# コース・レーティングとスロープ・レーティング — 難度の数学
**日付**: 2026-04-28
**分野**: golf
**タグ**: #course_rating #slope_rating #handicap #USGA #bogey_rating #数学的_handicap

## 学んだこと

### 1. なぜハンディキャップに「2 つの数値」が必要なのか

ゴルフのハンディキャップシステム (World Handicap System, WHS, 2020〜) は、**コースの難度を 2 つの独立した指標**で記述する:

- **Course Rating (CR)**: スクラッチプレーヤー (Handicap Index 0.0 のプレーヤー) が**そのコースで出すと予想されるスコア**。例: CR = 71.2 は、スクラッチが平均71.2打で回るコース。**絶対的な難度の基準点**
- **Slope Rating (SR)**: スクラッチとボギープレーヤー (HCI 20.0/24.0) のあいだで**コースが相対的にどれだけ難しくなるか**を示す指数。**相対的な難度の傾き**

この**「絶対基準 (CR)」と「相対傾き (SR)」**の二軸設計は、**1981 年にディーン・ノウルトン (Dean Knuth) が考案**した USGA の独自アプローチで、世界中のハンディキャップシステムが採用している。なぜ二軸が必要か:

- **CR だけ**だと、「スクラッチには簡単だがボギーには地獄」のコース (狭フェアウェイ・深ラフ・池絡み) と「両者ともにやさしい」コースを区別できない
- **SR だけ**だと、「絶対的な打数の重さ」が消える

両者を組み合わせることで、**「上手い人にも下手な人にも公平な調整」**が可能になる。

### 2. Course Rating の算出方法

Course Rating は**評価チーム (Allied Golf Association)** によって決定される。チームは 4–6 名のレーターが現地を訪れ、**スクラッチプレーヤー目線で各ホールを評価**する。

#### Yardage Rating (基礎)
まず**有効プレー長 (Effective Playing Length, EPL)** を計算:
- 公式長 (Yardage)
- ± 転がり (Roll, 平均10–15 yards 加算)
- ± 高低差 (Elevation, 上り下りで補正)
- ± 強制的なレイアップ (Forced Lay-Up)
- ± Doglegs (ドッグレッグでショートカット可否)

この EPL から **Yardage Rating = 基本長係数** を算出 (USGA テーブル参照、男性スクラッチ平均 220y 飛ばす前提)。

#### Obstacle Stroke Value
次に**10 のホール属性**を 0–10 のスケールで評価:
1. **Topography** (起伏)
2. **Fairway** (フェアウェイ幅)
3. **Green Target** (グリーン狙いの困難性)
4. **Recoverability and Rough** (リカバリーとラフ)
5. **Bunkers** (バンカー)
6. **Crossing Obstacles** (横切る障害物)
7. **Lateral Obstacles** (側方障害)
8. **Trees** (樹木)
9. **Green Surface** (グリーン面)
10. **Psychological Factors** (心理的要因)

各 0–10 評価に **重み係数**を掛け、**Obstacle Stroke Value (OSV)** を算出。

最終的に: **Course Rating = Yardage Rating + (Obstacle Stroke Value / 標準係数)**

例: 有効長 6,800y で Yardage Rating 70.4、OSV 補正 +1.2 → **CR = 71.6**

### 3. Bogey Rating と Slope の数式

評価チームは**同じプロセスをボギープレーヤー (HCI 20.0 男性 / 24.0 女性) 視点でやり直す**。**ボギーの飛距離は 200 y/180 y**、ボギーは**障害物の影響を 2–3 倍受ける**前提で OSV を再評価する。これにより **Bogey Rating (BR)** が算出される。

ここから **Slope Rating** を計算:

```
SR (men)   = (BR - CR) × 5.381  (小数切り上げ)
SR (women) = (BR - CR) × 4.240  (小数切り上げ)
```

定数 **5.381 / 4.240** はノウルトンが**全米の代表的コースで実測**した結果から逆算した係数。ボギーがスクラッチより**標準で 8 ストローク多く打てば SR = 113** になるよう正規化されている (8 × 5.381 ≈ 43.0 → 切り上げ 113 ではなく、実は SR=113 が「標準難度」になるように 5.381 が決定された)。

#### Slope Rating の範囲
- **最小**: 55 (極めて易しいコース、空港の練習場のような)
- **標準**: 113 (USGA基準の「平均」)
- **最大**: 155 (TPC Sawgrass、Pine Valley 等)

### 4. Course Handicap への変換

プレーヤーの **Handicap Index (HI)** からその日のコースで使う **Course Handicap (CH)** への変換:

```
Course Handicap = Handicap Index × (Slope Rating / 113) + (Course Rating - Par)
```

例: HI = 18.0 のプレーヤーが、CR = 72.0、SR = 130、Par = 72 のコースをプレーする場合:
```
CH = 18.0 × (130 / 113) + (72.0 - 72) = 18.0 × 1.150 + 0 = 20.7 → 21
```

つまり**「平均より難しいコース (SR=130 > 113)」では、HI 18 のプレーヤーは 21 ストローク貰える**。

WHS 2020 改定で **(CR - Par)** 項が追加され、**「par より難しいコース」**にも自動補正がかかるようになった。これは「**par 72 のコースで CR = 73.5 なら、par 自体が現実より易しい数字**」だからスクラッチですら +1.5 のハンディが必要、という理屈。

### 5. WHS の差分計算 (Score Differential)

ハンディキャップの更新には **Score Differential (SD)** を使う:

```
Score Differential = (113 / SR) × (Adjusted Gross Score - CR - PCC)
```

ここで:
- **Adjusted Gross Score**: ESC (Equitable Stroke Control) を適用したスコア
- **PCC (Playing Conditions Calculation)**: 当日のコース・コンディション補正 (-1 ~ +3)

過去 20 ラウンドのうち**最良 8 ラウンドの SD を平均**して **Handicap Index** とする。**「最良 8 / 20」**の選択は**「実力の上位 40%」**を取ることで偶発的好調を排除しつつ、**プレーヤーの「本来の実力」**を反映する設計。

### 6. PCC——日々変わる難度の動的補正

PCC は WHS 2020 で導入された**動的難度補正**。当日のコースで**全ハンデープレイヤーの平均スコアが事前予想より大きく外れた場合**、自動的に -1 〜 +3 で補正される。風速、ピン位置、グリーンスピード、雨で実難度が変わるためで、**「同じコースでも日によって難度が動く」**事実を統計的に取り込む。

これは USGA の**統計的決定**——個別プレイヤーの主観ではなく、**当日のラウンド結果集合から事後的に推定**する。

## 気づき・洞察

### 「相対化」の数学が公平を作る
Course Rating と Slope の二軸は、**プレーヤーの力量に応じてコースの難度が違って見える**という主観的事実を、**数値化された相対化**で取り扱う。これは**「絶対値より相対値」**という統計的発想——**標準偏差や正規化のテクニック**を社会的公平の道具として使う。Slope Rating の **「113 を標準」**にする正規化は、IQ テストの「100 を平均」とする設計と同型——**相対比較を絶対値の見かけで提示する**心理的設計。

### 5.381 の「神秘」——なぜこの係数か
SR の係数 5.381 は**完全な経験則**で、理論的に導出できる数値ではない。**全米の代表コースを調査して「この係数だと既存のハンディキャップ感覚と一致する」**点で決定された。これは**経験的較正 (empirical calibration)** の典型——理論ではなく**「現実との辻褄合わせ」**で定数を決める手法。物理学のフェルミ推定や、機械学習のハイパーパラメータ調整と本質的に同型。

### 「最良 8 ラウンドの平均」の哲学
WHS のハンディキャップが**「過去 20 のうち最良 8」**を取るのは、**「人間の実力は中央値ではなく上位確率分布で決まる」**という暗黙の哲学。**「悪い日もあるが、本来の実力に近いスコアを反映」**する設計で、これは**統計的には「上位四分位の代表値」**に近い。一方野球の打率が**「全打席の平均」**を取るのとは哲学が違う——**ゴルフは "ばらつきの大きい競技" であることが前提化**されている。

### PCC——ベイズ的補正の制度化
PCC は**当日のスコア分布から事後的に難度を推定する**仕組みで、**ベイズ更新の社会実装**と言える。事前確率 (=コースの公式 CR/SR) を、**当日の観察 (全プレーヤーのスコア)** で更新する。これは**金融市場のリアルタイムリスク調整**や**疾病サーベイランスの動的閾値**と同型——**静的な「公式値」と動的な「現実」の乖離を許容して逐次補正する**統計的成熟度の表現。

### 統計的公平 vs 競技的公正の緊張
ハンデキャップは**「ハンデ戦」では公平を作るが、「グロス競技」では意味を持たない**。プロツアーがハンデを使わないのは、**同等の上位プレーヤー間ではばらつきが小さく統計補正が不要**だから。これは**「公平」の概念が、母集団のばらつきの大きさによって異なる装置を要求する**ことを示す——アマチュアの社交的競技は統計的公平、プロの競技は実力ベースの絶対公正。

## 他分野との接続

- **golf (ストロークスゲインド・Pelz の17インチ理論)**: ストロークスゲインドが**個別ショットの寄与を統計的に分解**するのに対し、CR/SR は**コース全体の難度を統計的に分解**する——**統計的分解の階層**が違うだけで思想は同じ
- **golf (コース設計の戦略性)**: コースアーキテクトは **OSV を最大化する設計** (=戦略性) と **OSV を抑える設計** (=遊びやすさ) のあいだで判断する。SR は**設計判断の数値的フィードバック**
- **cs (Bloom filter の最適パラメータ)**: 偽陽性率 ε から最適 k を逆算するように、**目標 SR から OSV 配分を逆算**する——**目標値からのパラメータ最適化**
- **anatomy (筋膜のテンセグリティ)**: コースの 10 評価項目は**互いに独立しているように見えて相互補完的**——筋膜の張力ネットワークが**部分の総和でなく全体パターン**を作るのと類比的
- **piano (運指選択の最適化)**: 「最適運指」が手の物理的特性に依存するように、**Course Handicap が個人の HI に依存**する——**身体特性を入力としたパーソナライズ最適化**
- **bread (パン窯ごとの個性)**: 同じレシピでも窯ごとに**焼き上がりの実難度**が違うのと同型——**機械の個性を補正係数で吸収**
- **other (鳥の渡りのナビゲーション)**: 鳥が**地磁気・太陽・星・地形**の複数キューを統合してナビゲートするように、**ハンディキャップ計算は CR・SR・PCC・ESC の複数指標を統合**する
