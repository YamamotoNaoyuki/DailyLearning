## 学んだこと

### 6つのローカルシステムを1つに——WHS統合の経緯（2011→2020）

2020年1月以前、世界には**6つの異なるハンディキャップ計算系**が並立していた。USGA（北米）、CONGU（英国・アイルランド）、EGA（欧州大陸）、Golf Australia（豪州）、SAGA（南アフリカ）、AAG（アルゼンチン）の6統治機関。**約80カ国・1,500万人**が、それぞれ互換性のない数値でハンディを管理していた。同じ腕前の人でも、USGA系のHCP12がCONGU系では別の数字になる——国際的なクラブ対抗やリゾートでの競技で、公平な比較が物理的に不可能だった。

WHS構想は2011年、USGAとR&Aの会合に始まる。重要な発見は、**6つのシステムの「核となる原理」が実はほぼ同一**だったこと——いずれも「異なる技量のプレーヤーが合理的に対等な条件で競えるよう、プレーヤーの能力を測る」ことを目的としていた。違いは表現形式と細部のアルゴリズムだけ。そこで「最良の部分を選んで1つの仕様に統合する」標準化プロジェクトとして進行し、2020年1月に正式リリースされた。統治はUSGA + R&A、運用は各国・地域協会という**「統一フレームワーク + 局所運用」**の二層構造（[[ゴルフルール2019大改正と2023改正の哲学]]の Local Discretion と同型）。

### Course Rating と Slope Rating——2軸が表現する「難易度の関数」

WHSの土台は2つのレーティング。詳細な算出ロジックは[[コース・レーティングとスロープ・レーティングの数学]]に記録済みなので、ここでは**システム全体での役割**を中心に。

- **Course Rating (CR)**: スクラッチプレーヤー（HCP 0相当）がそのティーから出す**期待スコア**。par 72のコースでCR 73.5なら「上級者でも平均73.5打かかる」という絶対基準。
- **Slope Rating (SR)**: スクラッチとボギープレーヤー（HCP ≈ 20–24）の**スコア差の傾き**。標準113、範囲55–155。SRが高いほど「上級者には大したことなくても、平均的アマには罰が重い」コース構造を意味する。

直感的に言えば、CRはコース難易度関数の**切片**、SRは**傾き**である。難易度を「腕前に対する1次関数」として2パラメータで近似する——これがKnuthがUSGAで1981年に確立したモデルの本質。なぜ1次近似で足りるのか? それは「ハンディの目的は完璧な予測ではなく、異技量間の競技を成立させる程度の補正」だから。過剰なモデル化は運用コストと再現性を損なう（オッカムの剃刀の制度版）。

### Score Differential——「絶対スコア」を「相対実力」に変換する正規化

各ラウンドの成績は、生スコアではなく **Score Differential（スコアディファレンシャル, SD）** に変換される:

```
SD = (113 / Slope Rating) × (Adjusted Gross Score − Course Rating − PCC)
```

この式は1ラウンドを**「113という標準コースで打ったなら何打のオーバーか」に正規化**している。各項の意味:

- `AGS − CR`: 期待スコアからの乖離（生の出来）。
- `× 113/SR`: 傾きで割って標準コース基準にスケール変換。難しいコースでの80は易しいコースでの77より良い成績になりうる——その公平化がここで起きる。
- `− PCC`: その日の気象・セットアップによる難度変動を事後補正（後述）。

CSの目線で言えば、SDは**異なるベンチマーク環境で取った測定値を、共通の基準系に射影する正規化変換**。CRで原点合わせ、SRで尺度合わせ、PCCで当日ノイズ補正——測定値の標準化そのものである。

### Handicap Index = 直近20の上位8つの平均——「平均」ではなく「実力の上澄み」

20スコアが貯まると、Handicap Index (HI) は **直近20ラウンドのうちSDが良い上位8つの単純平均**で算出される（旧USGA系は「10/20 × 0.96」だった）。

設計思想上の最重要点: **HIは「平均的な出来」ではなく「demonstrated ability（実証された能力）」を測る**。20回中良い方8回だけを使うのは、「あなたが調子の良い日に出せる成績≒真の実力」という哲学。WHSは公式に「potential ability（潜在能力）」という旧来の言い回しを「demonstrated ability」へ言い換えた——測定可能で説明可能な「実証」概念へ寄せたわけだ。

統計的には、20サンプル中の**下位順序統計量を切り捨てた上側トリム平均**に相当する。なぜ上位40%か? ハンディの主用途は「ネット競技で対等にする」こと。もし全20回の平均をHIにすると、調子の良い日に誰もが自分のHIを大きく下回り、競技は「その日たまたま当たった人」が常勝する分散ゲームになる。**上位8つの平均にしておくと、各人が自分のHIに「届く」のは比較的良い日だけ**になり、ネットスコアの分布が選手間で揃う。つまり8/20は「能力の最良推定」ではなく**「競技公平性を最大化するパーセンタイルの経験的選択」**——SR公式の5.381同様、理論導出ではなく較正で決まった値だ。

### Course Handicap と Playing Handicap——HIから実プレーへの2段変換

ポータブルなHIは抽象量で、そのままでは何打もらえるか分からない。実プレーへは2段変換する:

```
Course Handicap (CH) = HI × (Slope Rating / 113) + (Course Rating − Par)
```

- `× SR/113`: 当日コースの傾きにHIを掛け戻す（標準化の逆変換）。
- `+ (CR − Par)`: WHS 2020で追加された項。**「par 72だがCR 73.5」のような、parとスクラッチ期待値のずれを吸収**する。これにより異なるティー間のCHが同一スケールで比較可能になり、家族で別ティーから回っても公平にネット競技できる。

さらに **Playing Handicap (PH)** = CH × ハンディ許容率（competition handicap allowance、例: ストロークプレー個人95%、フォアボール85%等）。CHが「コースに対する素の補正値」、PHが「その競技形式で実際に与え受けする打数」。形式ごとに分散特性が違うため、**形式に応じた割引率で公平性を再調整**する層が分離されている——インターフェイスと実装の分離に近い。

### Net Double Bogey——大叩きの上限（ESCの後継）

旧USGA系の Equitable Stroke Control (ESC) を置き換えたのが **Net Double Bogey (NDB)**。あるホールでハンディ計算上計上できる最大スコアは:

```
ホール上限 = par + 2 + そのホールで受けるハンディストローク数
```

たとえばHCP18のプレーヤーは全18ホールで1打ずつ受けるので、par 4のホールでは「4 + 2 + 1 = 7」がカウント上限。実際に9打叩いても、HI計算上は7として記録される（スコアカード上の実スコアは別。あくまでハンディ計算用の調整）。HI確立前の初回スコアでは一律 par + 5 が上限。

統計的役割は明快: HIは「上位8つの平均」なので大叩きホールは元々上位8つに残りにくいが、**単発の大叩きが20スコア中に紛れて分散を不当に膨らませるのを防ぐ外れ値クリッピング**。1ホールの惨事はそのプレーヤーの「能力」ではなく確率的事故——ロバスト統計でいうウィンザー化（極値を閾値に置換）の制度実装である。

### PCC（Playing Conditions Calculation）——当日難度のベイズ的事後補正

WHSの最も巧妙な統計装置が **PCC**。SD式の `− PCC` 項として効く。

- 各日終了時、そのコースで提出された全スコアを集計（**最低8スコア、HI 36.0以下のプレーヤー分のみ**有効）。
- 各プレーヤーの「期待スコア範囲」をHIから算出し、**実際に期待を上回った/下回った人数の割合**を見る。さらにコースのSR群から**SDの期待標準偏差**を計算し、観測された偏りが統計的に有意かを判定。
- 有意なら全員に補正を適用: **−1.0 / 0.0 / +1.0 / +2.0 / +3.0**。負は「易しく出た日」、正は「強風・速いグリーン等で難しく出た日」。

これは**「事前難度（CR/SR）を、当日の集団成績という観測で事後更新する」ベイズ更新の制度化**。CR/SRが事前分布、その日の全プレーヤースコアが尤度、PCC補正後のSDが事後推定。重要なのは**個人ではなく集団の分布を見る**こと——1人だけ崩れても天候のせいか自分のせいか分からないが、その日その場の全員が一様に2打高ければ「コンディションが難しかった」と帰属できる。母集団効果と個人効果の分離が、最低8スコア要件の根拠。

### Soft Cap / Hard Cap と Low Handicap Index——上方ブレーキ

ハンディが**上がる方向**には2段のブレーキがある。基準は **Low Handicap Index (Low HI)**: 直近の最新スコアから遡って **365日間の最低HI**（20スコア確立後に成立）。

- **Soft Cap**: 「8/20平均」がLow HIより**3.0打以上**高くなると、3.0を超えた分の上昇を**50%に減速**。
- **Hard Cap**: Soft Cap適用後も、上昇は**合計5.0打**で頭打ち（それ以上は一切上がらない）。

設計根拠は2つ。(1) **一時的な不調保護**: 「プレーヤーの真の能力が1年で大きく変わることはない」という前提のもと、数週間のスランプでHIが膨らみ、復調直後にネット競技で無双するのを防ぐ。(2) **公平性とサンドバッギング対策**: もしキャップがなければ、大きく上がった人ほど「自分のHIを下回る」確率が高くなり、一貫したプレーヤーより有利になる。キャップは**HIが上がりやすい方向（=競技で有利になる方向）の動きだけを抑える非対称設計**——下げる方向には制限がない。これが核心: WHSの上方ブレーキは全て**「人は自分が有利になる方向に偏る」という行動バイアスへの対抗装置**だ。

### Exceptional Score Reduction——下方への即時補正

逆に**異常に良いスコア**には即時の下方補正が入る:

- SDがその時点のHIより**7.0打以上良い** → 直近20スコア全てに **−1.0** を適用。
- SDが**10.0打以上良い** → **−2.0** を適用。

これにより、突発的な好スコア1本でHIが即座に下方修正される。以後の通常スコアにはこの補正は乗らないので、効果は20スコアの入れ替えとともに**自然に減衰**する。「1回だけのまぐれで永久にハンディを操作する」ことを防ぐ、時間減衰付きの罰則項である。

### サンドバッギングとの統計戦争——WHSの本質的設計目標

ゴルフは**プレーヤー自身がスコアを申告する**自己申告制スポーツ。ここで構造的に不正インセンティブが生じる——競技賞品を得るために実力より高いHIを維持する**サンドバッギング**だ。WHSの各装置は、この行動バイアスへの多層防御として読み解ける:

| 装置 | 防ぐバイアス・不正 |
|------|------|
| 上位8/20平均 | 悪いスコアの選択的提出による水増し（悪い回は元々除外） |
| Net Double Bogey | 単発大叩きでHIを膨らませる外れ値操作 |
| PCC | 「難しい日のスコアだけ提出」による環境操作 |
| Soft/Hard Cap | スランプを装った計画的上方インフレ |
| Exceptional Score Reduction | 好スコアの即時反映を遅らせる(=本当の実力を隠す)操作 |
| Handicap Review（委員会裁量） | 上記をすり抜ける異常パターンの人手検出 |

USGA/R&Aの設計思想は一貫している——**「公平性は性善説では守れない。インセンティブ構造に逆らうバイアスへのカウンター項を、計算式そのものに埋め込む」**。これはメカニズムデザイン（耐戦略性, strategy-proofness）の発想に近い。

### 日本——JGAハンディキャップのWHS移行（J-sys）

日本ゴルフ協会(JGA)は2021年4月に専用サイトを開設し、**2022年4月にWHSを国内本格適用**した。JGAが運営するHI算定・管理システム **J-sys**（Glidとも連携）をWHS仕様へ移行——画面構成はほぼ維持しつつ、内部計算をWHSルールに差し替えた。これにより日本のゴルファーのHIが国際的に通用する数値になり、海外競技や訪日ゴルファーとの公平な比較が可能になった。JGAは取得率向上と参加コース拡大を移行目的に掲げている。

## 気づき・洞察

- **WHSは「測定値の標準化パイプライン」そのものである。** 生スコア → (CR/SRで正規化) → SD → (PCCで当日ノイズ除去) → (上位8つのトリム平均) → (Cap/ESRで外れ値・バイアス補正) → HI → (SR/113で逆変換) → CH → (許容率で形式調整) → PH。各段が「異なる環境・条件で取られた測定値を、共通基準系の比較可能量へ変換する」という単一目的に奉仕する。これは実験データの前処理パイプライン（正規化→外れ値除去→ベイズ補正）と構造的に同型。

- **すべての非対称設計は「人は自分が有利になる方へ偏る」への対抗**。Cap は上方向のみ、ESR は下方向のみ。NDB は大叩きのみクリップ。PCC は集団分布で個人の言い訳を封じる。WHSの数式は単なる計算規則ではなく、**自己申告制という制度的弱点に対して書かれた耐戦略的メカニズム**だ。Goodhart の法則（指標が目標になると壊れる）に、計算式の中で先回りして応答している。

- **「平均ではなく上位8つ」が意味するもの。** これはHIを「能力の点推定」だと思うと理解できない。WHSが最適化しているのは個人の推定精度ではなく**選手間のネットスコア分布の整列**——競技公平性という集団量だ。だから8/20も113もPCC閾値も「理論的最適」ではなく「現場で公平が成立する較正値」。**システムの目的関数を取り違えると設計が読めない**典型例。

- **6システム統合は「コア原理が同一だった」から成功した。** 表現形式の違いに惑わされず本質の一致を見抜き、最良部分を選んで1仕様に畳む——これは異なる実装を1つの標準仕様に統合するソフトウェアの標準化（複数方言→共通規格）と完全に同じ運動。違いがUIで一致が本質、という構造の発見が統合の鍵だった。

- **[[DECADEシステムとScott_Fawcettの統計的コースマネジメント]]**: DECADEがプレーヤー個人の期待値最適化なら、WHSは協会による集団公平性最適化。**同じ統計思考の攻守両面**。さらにNDBは「大叩きが指標を壊さない」点でDECADEの「ボギーを受容しダブルを避ける」哲学とスコア構造観を共有する。

- **[[メジャー選手権のコースセットアップ哲学]]**: PCCが補正しようとする「当日の難度変動」は、まさにセットアップ（ティー位置・ピン・ラフ・グリーンスピード）が生む可変量。USオープンの過酷セットアップの日はPCCが+方向に振れるはず。[[Stimpmeterとグリーンスピードの科学]]のグリーン速度変動も、PCCが集団スコアを通じて間接的に捕捉している。

- **CS/統計**: 上位8つの平均=トリム平均、NDB=ウィンザー化、PCC=ベイズ事後更新、Cap/ESR=非対称ペナルティ項、全体=耐戦略的メカニズムデザイン。**統計的ロバスト推定とメカニズムデザインの実用標本**としてゴルフ規則は教材的に優れている。

## 次に深掘りしたいこと

- **PCCの正確な閾値テーブル**——「期待を上回った人数割合がどの値で−1/+1/+2/+3に振れるか」の具体的判定境界（Rules of Handicapping Rule 5.6の付属表）。SDの期待標準偏差をSR群からどう合成するかの数式詳細。
- **Cap適用の数値例**——Low HI、8/20平均、Soft/Hard Capが相互作用する具体ケースを手計算で追い、50%減速が連続値でどう効くか確認したい。
- **DECADE的にHIを「ゲーム」できるか**——理論上HIを最大限有利に保つ提出戦略は存在するか、そしてWHSの各装置がそれをどこまで封じているか（メカニズムの耐戦略性の限界）。
- **9ホールスコアのWHS取り扱い**（2024改正でExpected Scoreを使う方式に変更）——18ホール前提の統計設計に9ホールをどう接合するか。
- **日本固有の論点**——J-sysのコースレーティング整備状況、ローカルルールの扱い、HI取得率の国際比較。
