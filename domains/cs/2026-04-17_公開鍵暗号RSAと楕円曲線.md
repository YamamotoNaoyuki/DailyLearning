# 公開鍵暗号 — RSAと楕円曲線暗号の数学的基礎

**日付**: 2026-04-17
**分野**: cs
**タグ**: #公開鍵暗号 #RSA #ECC #ECDLP #Curve25519 #PKI

## 学んだこと

### 公開鍵暗号の発明史
1976年、Diffie & Hellman が「New Directions in Cryptography」で、共有秘密なしに鍵を交換する方法を提案。一方向関数の概念を確立。1977年、Rivest・Shamir・Adleman が **RSA** を発表（特許: 1983-2000、米国）。1985年、Koblitz と Miller が独立に **楕円曲線暗号（ECC）** を提案。両者とも、対応する離散対数問題（DLP）または素因数分解問題（IFP）の計算困難性を安全性の根拠とする。

### RSAの数学的構造
**鍵生成**:
1. 大きな素数 p, q を選び N = pq を計算（典型的に 2048-4096 ビット）
2. φ(N) = (p-1)(q-1)（オイラー関数）を計算
3. gcd(e, φ(N)) = 1 を満たす公開指数 e を選ぶ（典型的に e = 65537）
4. 秘密指数 d = e⁻¹ mod φ(N) を計算（拡張ユークリッド互除法）
5. 公開鍵: (N, e)、秘密鍵: d（および分解 p, q）

**暗号化**: c = m^e mod N
**復号**: m = c^d mod N

正当性は **Eulerの定理** m^φ(N) ≡ 1 (mod N) と **中国剰余定理（CRT）** に基づく。CRT 復号で約4倍高速化。

**安全性**: 整数因数分解（IFP）の困難性。最良の汎用アルゴリズムは GNFS（一般数体ふるい法、準指数時間 L_N[1/3, c]）。RSA-829（2048ビット弱）まで分解実績はあるが、RSA-2048 はまだ未達。NIST は 2030年以降の RSA-2048 廃止を勧告。

### 楕円曲線暗号（ECC）の数学
**楕円曲線**: 体 K 上の方程式 y² = x³ + ax + b（4a³ + 27b² ≠ 0）で定義される非特異曲線。「無限遠点 O」を加えて群を成す。

**点の加法**: 幾何学的には「2点を結ぶ直線が曲線と交わる第3の点を x 軸対称した点」。代数的には傾き λ を計算し、(x_R, y_R) を導出。「点の k 倍算」 P → kP は、加算と倍化の繰り返し（Double-and-add法、O(log k) 演算）。

**有限体上の楕円曲線**: 暗号応用では F_p（素数体）または F_{2^m}（二進体）上で定義。F_p 上の点の個数は Hasse の定理により |#E(F_p) - (p+1)| ≤ 2√p。

**ECDLP（楕円曲線離散対数問題）**: 「点 G と P = kG が与えられたとき、k を求めよ」が困難。最良の汎用アルゴリズムは Pollard's rho（O(√n)、n は群の位数）。指数より大幅に高速な手法は知られていない。

### ECCの優位性
| 安全性レベル | RSA鍵長 | ECC鍵長 | 比 |
|----------|--------|--------|---|
| 80ビット | 1024 | 160 | 6.4x |
| 112 | 2048 | 224 | 9.1x |
| 128 | 3072 | 256 | 12x |
| 192 | 7680 | 384 | 20x |
| 256 | 15360 | 512 | 30x |

ECC は同等安全性で約 1/10 の鍵長。署名・暗号化処理も大幅に高速。低電力デバイス（IoT・スマートカード）・帯域制約環境で必須。

### 主要曲線の比較
- **NIST P-256（secp256r1）**: NIST/SECG 標準。WebAuthn・TLS の主流。係数の選択にバックドア疑惑（NSA関与の歴史）
- **secp256k1**: y² = x³ + 7（特殊化された Koblitz 曲線）。Bitcoin・Ethereum で採用。明示的に「nothing up my sleeve」で構築
- **Curve25519** (Daniel J. Bernstein, 2005): モンゴメリ曲線 y² = x³ + 486662x² + x、素数 p = 2²⁵⁵ - 19。**安全性・性能・実装シンプルさのバランスで最良**とされる。Signal、TLS 1.3、SSH、WireGuard、Tor で広く採用
- **Ed25519**: Curve25519 と双有理同値の Twisted Edwards 曲線 edwards25519 上の **EdDSA** 署名スキーム。決定論的署名（乱数源不要）でフォールトトレランス高い。OpenSSH のデフォルト

### 安全な実装の難しさ
**サイドチャネル攻撃**: 計算時間・電力消費・電磁波からの鍵漏洩。RSA は CRT 復号の操作時間差（Bleichenbacher 攻撃）、ECC は条件分岐（unsafe な double-and-add 実装）が脆弱。
- 対策: 定数時間実装（constant-time）、Montgomery ladder、scalar blinding
- Curve25519 は「設計時から定数時間実装が容易になるよう」選ばれており、実装ミスへの耐性が高い

**乱数生成の重要性**: ECDSA の k 値（per-message nonce）の再使用は秘密鍵を完全に漏洩する。Sony PlayStation 3（2010）は ECDSA k を固定値で使い完全クラックされた。Ed25519 は決定論的でこのリスクを構造的に排除。

### 量子コンピュータの脅威
**Shor のアルゴリズム**（1994）は量子コンピュータで多項式時間に IFP も DLP も解く。「十分大きな量子コンピュータ」が実現すると RSA・ECC ともに即座に破られる。
- **ハイブリッド暗号**（Hybrid PQC）: NIST が 2024年に標準化した **ML-KEM**（旧 CRYSTALS-Kyber、鍵交換）、**ML-DSA**（旧 CRYSTALS-Dilithium、署名）、**SLH-DSA**（旧 SPHINCS+、署名）。格子暗号・ハッシュ署名ベース
- 「Harvest Now, Decrypt Later」攻撃: 現在通信を保存し将来量子コンピュータで復号する戦略。長期機密データには既に PQC 対応が必要

## 気づき・洞察

**RSA から ECC への移行は「数学的精緻さ」と「実装の責任」のトレードオフ**: RSA は数論の古典的問題（素因数分解）に基づき、概念が直感的で教育に適する。一方 ECC は群論・代数幾何の高度な理論を要求し、実装が複雑化する。しかし鍵長効率（1/10）と性能優位は決定的で、現代の暗号インフラ（TLS 1.3、WebAuthn、Bitcoin）は事実上 ECC 中心に再構築された。「数学的洗練が技術競争を勝ち抜く」典型例。

**Curve25519 設計の哲学的意義**: Bernstein が Curve25519 を設計した動機は「NIST 曲線への不信」だった。NSA 関与の疑惑、サイドチャネル耐性の弱さ、実装ミスの誘発。Curve25519 は「あらゆる入力で定数時間実装可能」「nothing up my sleeve」「scalar の検証が不要」など、**実装者を信用しない設計**を貫いた。これは「Cryptographic primitives should be foolproof」という現代暗号工学の合意を形成する転換点となった。「数学的に安全」より「現実に安全」を優先する設計哲学。

**Post-Quantum への移行は「不可逆な未来計画」**: 量子コンピュータは「いつ実用化するか不明」だが、「実用化したら即座に既存暗号が崩壊する」。この非対称性ゆえ、業界は 2026年現在「PQC 移行は 10年がかり」のロードマップを描く。NIST PQC 標準（2024年8月最終化）の採用が始まったばかりで、TLS のハイブリッド鍵交換（X25519 + ML-KEM）が Cloudflare・Google で 2024-2025 に展開された。**暗号は「変えなくていい」が一番安全だが、量子はその前提を破壊する**——20年に一度の暗号インフラ大転換が始まっている。

## 次に深掘りしたいこと
- **格子ベース暗号（Lattice-based）の数学**: LWE（Learning With Errors）と Module-LWE の構造、ML-KEM/ML-DSA の内部
- **ペアリング暗号（Pairing-based）**: BLS 署名、Identity-based encryption、ゼロ知識証明（zk-SNARKs）の数学的基礎
- **同型暗号（Homomorphic Encryption）**: BFV/CKKS スキーム、暗号化されたデータ上での演算
- **TLS 1.3 のハンドシェイク詳細**: ECDHE_X25519 + AEAD-ChaCha20-Poly1305 の組み合わせがどう構成されるか
- **量子鍵配送（QKD）**: BB84プロトコル、無条件安全性の物理的根拠
