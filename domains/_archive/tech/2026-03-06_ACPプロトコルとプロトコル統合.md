# ACP（Agent Communication Protocol）とプロトコル統合の動向
**日付**: 2026-03-06
**分野**: テクノロジー
**タグ**: #ACP #BeeAI #IBM #A2A #MCP #プロトコル統合

## 学んだこと

### ACPとは何か

Agent Communication Protocol（ACP）は、2025年5月にIBM Researchが発表したオープンプロトコル。AIエージェント間の通信を標準化し、フレームワーク・技術・組織の壁を超えた協調を可能にする。目標は「エージェント通信のHTTPを作る」こと。

### BeeAIプラットフォーム

ACPはIBMの**BeeAI**プラットフォームの通信基盤として開発された。BeeAIは2025年3月にLinux Foundationに寄贈されたオープンソースプラットフォームで、AIエージェントの発見・実行・合成（composing）を行う。

- **BeeAI Framework**: Python/TypeScript対応のプロダクショングレードフレームワーク
- **BeeAI Platform**: ACP準拠エージェントの発見・共有プラットフォーム

### 技術的特徴

ACPはA2Aと同様に既存の標準の上に構築：

- **JSON-RPC over HTTP/WebSockets**: 通信プロトコル
- **小さなコントロールプレーン**: エージェントの発見、認証、協調を管理
- **ステートレス設計**: セッション管理なしで軽量に動作可能

### 3プロトコルの位置づけ——MCP, A2A, ACP

| 特性 | MCP (Anthropic) | A2A (Google) | ACP (IBM) |
|------|-----------------|--------------|-----------|
| 方向 | 垂直（エージェント↔ツール） | 水平（エージェント↔エージェント） | 水平（エージェント↔エージェント） |
| 焦点 | コンテキストとツール接続 | タスク委任とケイパビリティ発見 | マルチエージェント合成と発見 |
| 発見 | ツールカタログ | Agent Card | エージェントレジストリ |
| 成熟度 | 最も成熟。広く採用 | v0.3。150+組織 | pre-alpha（発表時） |
| ガバナンス | AAIF (Linux Foundation) | Linux Foundation | Linux Foundation |

### プロトコル統合の衝撃的展開

**2025年9月、ACPチームとA2Aチームが統合を発表。** 両チームは統一標準の共同開発を開始した。

DeepLearning.AIのACPコースは2026年2月4日にA2Aプロトコルのコースに置き換えられた。これは事実上、ACPがA2Aに合流したことを意味する。

この統合の背景：
- A2Aは150+組織のサポートと先行するエコシステム
- ACPはIBMの技術的深さとBeeAIのプラットフォーム
- 分裂したプロトコルは採用の障壁（MCPの成功は「1つの標準」だったからこそ）
- Linux Foundation傘下で統一ガバナンスが可能に

### 現在のプロトコル地図（2026年3月時点）

```
エージェント内部（垂直）: MCP（Anthropic → AAIF）
  ↕
エージェント間（水平）: A2A（Google + IBM/ACP統合 → Linux Foundation）
```

かつての「3プロトコル問題」は、MCP（垂直）+ A2A（水平）の**2軸体制**に収斂しつつある。

## 気づき・洞察

プロトコルの統合過程は、技術史上の標準化戦争のパターンを踏襲している。VHS vs ベータ、Blu-ray vs HD DVD、USB-C vs Lightning——勝者は技術的優位性よりも、エコシステムの広さとガバナンスの開放性で決まる。A2AがACPを吸収できたのは、150+組織というエコシステムの厚みと、Linux Foundation傘下のオープンガバナンスによる。

MCPとA2Aの「2軸体制」は、前回学んだ「垂直（ツール接続）vs 水平（エージェント間）」の補完関係が、市場によって追認された形だ。「何でもできる1つの万能プロトコル」ではなく、「それぞれのレイヤーに特化した2つのプロトコル」が生き残る——これはOSI参照モデルの思想と同じだ。

## 他分野との接続

- **SDV**: CAN bus→Automotive Ethernet→SOME/IPの進化も、プロトコル統合の歴史。車載通信もやがて1-2の支配的プロトコルに収斂するだろう
- **音楽**: メシアンの「移調の限られた旋法」は7種。ACPの消滅は「旋法が多すぎると言語として機能しない」ことの証明。言語（プロトコル）は限定されることで力を持つ
- **哲学**: 道元の正法眼蔵が曹洞宗という「1つの系譜」に収斂したように、プロトコルも淘汰を経て統一へ向かう。多様性の段階→収斂の段階は普遍的パターン
