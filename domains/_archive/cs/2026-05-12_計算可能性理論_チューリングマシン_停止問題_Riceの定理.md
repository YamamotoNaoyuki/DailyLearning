## 学んだこと

### なぜ「計算とは何か」を厳密に定義する必要があったか

1930 年代、ヒルベルトの **Entscheidungsproblem（決定問題）**——「与えられた一階述語論理式が証明可能かを機械的に判定する手続きはあるか？」——に答えるには、「機械的手続き（effective procedure）」を数学的に定義する必要があった。チャーチ（λ計算）、ゲーデル（再帰関数）、チューリング（チューリングマシン）が独立に定式化し、**すべて等価**であることが証明された。これが **チャーチ・チューリングのテーゼ**：「直感的に計算可能 ⟺ チューリングマシンで計算可能」。これは数学的定理ではなく**自然法則的な主張**（直感的概念と形式的概念の同一視）だが、90 年以上反例がなく、量子計算ですら「計算可能なものの範囲」は変えない（変えるのは効率＝計算量だけ）。

チューリングマシン（TM）の本質的要素は：(1) 無限のテープ（記憶）、(2) 有限の状態（プログラム）、(3) 局所的な書き換え規則。重要なのは **「無限のテープ × 有限の制御」** という非対称——プログラムは有限なのに、扱えるデータは無限。この組合せが「万能性（universality）」を生む。

### 万能チューリングマシン——「プログラム＝データ」の発見

チューリングの決定的洞察：**ある TM の記述（遷移規則の表）を文字列として符号化し、それを別の TM の入力テープに置ける**。**万能チューリングマシン U** は、入力として「TM M の符号 ⟨M⟩」と「M への入力 w」を受け取り、M が w 上で振る舞う通りにシミュレートする。U(⟨M⟩, w) = M(w)。

これが現代の**ストアドプログラム方式（フォン・ノイマン型コンピュータ）の理論的原型**——「プログラムはメモリ上のデータに過ぎず、別のプログラム（OS、インタプリタ、コンパイラ、仮想機械、eval）が読んで実行できる」。Lisp の `eval`、JVM、QEMU、ブラウザの JS エンジン、そして MoE モデルの重みを読んで走らせる推論エンジン——すべて万能 TM のインスタンス。「コードとデータの区別は便宜的なもので、本質的なものではない」というのが計算理論の出発点であり、同時に多くのセキュリティ脆弱性（コードインジェクション、バッファオーバーフロー）の根源でもある。

### 停止問題（Halting Problem）——対角線論法の本領

**問**：「TM M と入力 w が与えられたとき、M が w 上で停止するか（無限ループしないか）を判定する TM H は存在するか？」
**答**：存在しない（決定不能 undecidable）。

証明（対角線論法）：仮に H(⟨M⟩, w) が「M が w で停止するなら yes、しなければ no」を必ず有限時間で返すとする。すると次の TM D を作れる：
```
D(⟨M⟩):
  if H(⟨M⟩, ⟨M⟩) == yes:   # M が「自分自身の符号」を入力に取って停止するなら
      loop forever            # わざと無限ループ
  else:
      halt                    # 停止
```
ここで D に「D 自身の符号 ⟨D⟩」を入力として与える：D(⟨D⟩) は？
- もし D(⟨D⟩) が停止するなら、定義により H(⟨D⟩,⟨D⟩)=yes だったはず → だが then 節は loop forever → 停止しない。矛盾。
- もし D(⟨D⟩) が停止しないなら、H(⟨D⟩,⟨D⟩)=no → else 節 → halt → 停止する。矛盾。

どちらも矛盾。ゆえに H は存在し得ない。**「自己言及＋否定」が古典的パラドックス（嘘つきのパラドックス、ラッセルのパラドックス、カントールの対角線論法）と同じ構造**で、計算という文脈でそれが「決定不能性」として現れる。

「停止性は決定不能」が意味するのは——**完璧な「無限ループ検出器」「最適化コンパイラの停止保証」「リソース上限の事前判定」は原理的に作れない**。実務では「ほとんどのケースで判定できる近似」（タイムアウト、停止性解析の保守的近似）で凌ぐ。

### Rice の定理——「意味のあるプログラム解析はすべて決定不能」

停止問題の一般化であり、**実務に最も効く一撃**。Rice の定理（1953）：

> **プログラムが計算する関数（部分関数）についての、自明でない（trivial でない）あらゆる意味的（semantic）性質は決定不能である。**

「自明でない」＝「その性質を持つプログラムも持たないプログラムも存在する」。「意味的（extensional）」＝「プログラムの**入出力の振る舞い**だけで決まる性質」（構文的な性質——「ソースが 100 行以下か」「`goto` を含むか——ではなく）。

帰結（すべて決定不能）：
- このプログラムは恒等関数を計算するか？
- このプログラムは常に 0 を返すか？
- 二つのプログラムは同じ関数を計算するか？（**プログラム等価性判定**——コンパイラ最適化の正しさ検証が原理的に自動化できない理由）
- このプログラムはすべての入力で停止するか？（**全域性 totality**）
- このプログラムはエラー（ゼロ除算、NULL 参照）を起こす入力があるか？（**完全な静的バグ検出は不可能**）
- このコードはマルウェアか（特定の悪性挙動を示すか）？（**完全なウイルス検出は不可能**——シグネチャや振る舞いヒューリスティクスに頼る理由）

Rice の定理が言っているのは「**プログラムを実行せずに、その"意味"を完全に知ることはできない**」——プログラムの意味は、それを（あらゆる入力で）走らせること以外に確定する方法がない。これが静的解析ツール（型チェッカ、リンタ、定理証明器、抽象解釈）が**必ず**「保守的（soundness のために誤検出を許す）」または「不完全（false negative を許す）」になる根本理由。

### 決定不能性の「強さ」を測る——帰着と次数

決定不能問題にも階層がある。**多対一帰着（many-one reduction）** A ≤ₘ B：「A の問題例を B の問題例に変換する計算可能関数があり、答えが一致する」。A ≤ₘ B かつ B が決定可能なら A も決定可能——だから「B は A 以上に難しい」。停止問題は **半決定可能（recursively enumerable, r.e.）**——「停止する」ケースは（実際に走らせれば）有限時間で yes と分かるが、「停止しない」ケースは確認できない。その**補集合（停止しない問題）は r.e. ですらない**。さらに難しい問題（停止問題のオラクルを使っても解けない問題＝「停止問題の停止問題」…）が無限に積み重なり、これが **算術的階層（arithmetical hierarchy）** Σₙ / Πₙ。「∀x ∃y ...」のような論理式の量化子の交替数が、その述語の「計算不能性の度合い」に対応する——ロジックと計算が再び繋がる（[[2026-05-01_Curry_Howard同型と命題型対応]] の延長線上）。

### ゲーデルとの関係——不完全性は決定不能性の影

ゲーデルの第一不完全性定理（「十分強い無矛盾な公理系には、真だが証明できない命題が存在する」）は、計算可能性の言葉で見通しよく言い換えられる：もし「論理式 φ が証明可能か」が決定可能なら、「TM M が停止するか」も決定可能になってしまう（停止は算術の命題として書ける）——だが停止問題は決定不能。ゆえに証明可能性は決定不能（Entscheidungsproblem への否定的回答＝チャーチとチューリングが 1936 に独立に示した）。**「証明できる」⊊「真である」のギャップは、「決定できる」⊊「半決定できる」のギャップの論理版**。

### 「決定不能を回避する」——制限という工学

理論が「不可能」と言ったら、工学は「では何を諦めればできるか」を問う。これが現代システムの至る所にある設計判断：
- **eBPF の verifier**（[[2026-05-08_eBPFとLinuxカーネル可観測性]]、tech 既出）：任意プログラムの停止性判定は不可能 → だから eBPF は**言語自体を制限**（無制限ループ禁止／bounded loops のみ、命令数上限、有界スタック）して、停止性を**決定可能な部分集合**に押し込めた。「決定不能な問題は、入力を狭めれば決定可能になる」。
- **型システム**：[[2026-04-24_Hindley_Milner型推論とlet多相]] の HM 型推論は決定可能、しかし System F（全称型の完全な型推論）は決定不能、依存型の型検査も一般には決定不能 → 言語設計者は「表現力 vs 判定可能性」を綱引きする。
- **正規表現と文脈自由文法**：正規言語の包含・等価は決定可能、文脈自由言語の等価性は決定不能 → だから「正規表現の最適化」はできても「任意パーサの等価性チェック」はできない。
- **モデル検査**：有限状態に限れば（あるいは抽象化して有限化すれば）性質検証は決定可能 → だからハードウェア検証や有限状態プロトコル検証は機械化できるが、無限状態系では半自動・近似に頼る。
- **全関数型言語（Agda, Idris の totality checker）**：「停止しない関数」を書けないように構文を制限（構造的再帰のみ許す）→ チューリング完全性を犠牲にして「すべての関数が停止する」保証を得る。

つまり **「チューリング完全性」は両刃**——万能だが解析不能。実用システムの多くは「ほぼ万能だが解析可能な部分集合」を慎重に選んでいる。設定ファイル形式が（うっかり）チューリング完全になると「設定の検証ができなくなる」（実話：sendmail 設定、一部の CSS/正規表現は意図せず計算能力が高すぎる）。

## 気づき・洞察

計算可能性理論の核は **「自己言及は両刃の剣」** という一点に尽きる。「プログラムをデータとして扱える（万能 TM、eval、メタプログラミング）」という強力さと、「プログラムについての完全な判定はできない（停止問題、Rice）」という限界は、**同じコイン（万能性＝universality）の表裏**。柔軟性のコストとして、自己分析の不可能性を払っている。これは脳が「自分自身を完全に理解できるか」という問いとも響くし、ゲーデル＝エッシャー＝バッハ的な「自己参照ループ」の主題そのもの。

実務的に最も重要な悟りは——**「完璧な静的解析が存在しないのは、ツールが未熟だからではなく、原理的に不可能だから」**。だから「型チェッカが通ったのにバグがある」「リンタが見逃した」「ウイルス対策をすり抜けられた」は、努力不足ではなく Rice の定理の必然。ゆえに正しい戦略は「完璧を目指す」のではなく「**どの近似（保守的 or 楽観的）を選ぶか、誤検出と見逃しのトレードオフをどう設計するか**」——これは tech の MoE 負荷分散（「最適ルーティングは難しいから学習で近似」）、anatomy の把持力制御（「最適握力は計算できないから安全マージンで近似」）、golf のコースマネジメント（「完璧なショットは無理だから期待値で意思決定」）と完全に同じ思考様式。**「理論が不可能と言う ⇒ 入力を制限する／近似で妥協する／トレードオフを明示的に設計する」——これがシステム工学の最も汎用的なパターン**。

そしてもう一つ：「無限のテープ × 有限の制御」という非対称が万能性の源、という構造は美しい。有限の DNA（制御）が無限に多様な環境（テープ）に対応する生物、有限のオーケストレーション（指揮者）が無限の音楽的状況に対応する演奏（[[ブラームス4番]]）、有限のルーター（MoE）が無限の入力に対応するモデル——「有限の規則 × 無限の状況」という普遍構造の、最も純粋な数学的表現がチューリングマシンだ。

## 他分野との接続

- **tech（今日のエントリ・MoE）**: ルーティングという組合せ最適化を「学習で近似」するのは、「決定不能／NP困難な問題を実用的近似で凌ぐ」の典型。また「重みを読んで走らせる推論エンジン」は万能 TM のインスタンス。
- **cs 内・[[2026-04-26_NP完全性とCook_Levin定理]]**: 決定不能（できない）と NP完全（できるが遅い）は別の壁——計算理論は「解けるか（computability）」と「速く解けるか（complexity）」の二層構造。本エントリは下の層（より根源的な不可能性）。
- **cs 内・[[2026-05-01_Curry_Howard同型と命題型対応]] / [[2026-05-04_SSA形式とコンパイラ最適化]]**: 型推論の判定可能性、コンパイラ最適化の正しさ検証——Rice の定理が「どこまで自動化できるか」の上限を引く。SSA も「保守的に近似された」データフロー解析。
- **解剖学（今日のエントリ・能動的触知覚）**: 「物の性質は、能動的に探索する（走らせる）以外に確定できない」——プログラムの意味は実行する以外に確定できない、と構造が同じ。観測には介入が要る。
- **other（ゲーム理論）**: ナッシュ均衡の計算（一般には PPAD 困難）、無限ゲームの解の存在——ゲーム理論にも「存在するが構成的に求まらない／効率的に求まらない」という計算可能性・計算量の壁がある。
- **music（ブラームスの developing variation）/ piano（ポリリズム）**: 「有限の素材（8 小節の主題／2 つのリズム）を無限に変容させる」——有限の規則から無限の多様性を生む、万能 TM と同じ生成原理。

## 次に深掘りしたいこと

- **再帰定理（Kleene's recursion theorem）**：「任意のプログラムは自分自身の符号を取得できる」——quine（自己出力プログラム）やコンピュータウイルスの理論的基盤、不動点定理との関係
- **Busy Beaver 関数 Σ(n)**：n 状態 TM が停止する前に書ける最大の 1 の数——計算可能でない（停止問題に帰着）、しかし小さい n では具体値が計算され、BB(5)=47,176,870 が 2024 に確定、BB(6) は天文学的でゲーデル独立の可能性も
- **オラクルマシンと相対化**：P vs NP 問題が「相対化する技法では解けない」（Baker–Gill–Solovay）——証明技法の限界そのものを論じるメタ理論
- **Chaitin の Ω（停止確率）**：アルゴリズム的にランダムな実数、コルモゴロフ複雑性と決定不能性の交点
- **超計算（hypercomputation）**：チャーチ・チューリングのテーゼに「物理的に」反例はあり得るか（無限精度の実数計算、相対論的計算、量子重力…）——ほぼ全て否定的だが境界線の議論として面白い
