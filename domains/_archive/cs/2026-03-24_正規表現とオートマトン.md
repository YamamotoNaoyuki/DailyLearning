# 正規表現とオートマトン
**日付**: 2026-03-24
**分野**: コンピュータサイエンス（計算理論）
**タグ**: #正規表現 #オートマトン #Kleene定理 #Chomsky階層 #コンパイラ #字句解析

## 学んだこと

### Kleeneの定理: 三つの等価な表現

正規言語には3つの等価な記述方法がある:
- **正規表現（RE）**: 言語の宣言的記述（「何を受理するか」）
- **NFA**: 非決定的な認識モデル（分岐を許す計算）
- **DFA**: 決定的な認識モデル（一意な計算パス）

Kleeneの定理（1956）はこの三者の表現力が完全に等価であることを示す。証明は3つの構成的変換で成り立つ。

#### Thompson's Construction: RE → NFA

Ken Thompsonが1968年の論文 "Regular Expression Search Algorithm" で提示。正規表現の構文木を再帰的にNFAの断片に変換する。

設計上の核心: 各部分NFAが**単一の開始状態と単一の受理状態**を持つという不変条件。これにより和（union）・連接（concatenation）・Kleene閉包の合成がε遷移による機械的な接続で済む。

- 基本要素: 文字 `a` → 開始状態 --a--> 受理状態
- 和 `R|S`: 新開始状態からR, Sの開始状態へε遷移。R, Sの受理状態から新受理状態へε遷移
- 連接 `RS`: Rの受理状態からSの開始状態へε遷移
- 閉包 `R*`: 新開始状態からRの開始状態と新受理状態へε遷移。Rの受理状態からRの開始状態と新受理状態へε遷移

生成されるNFAの状態数は正規表現のサイズ `m` に対して **O(m)** — これが後述のThompson NFAシミュレーションの効率の根拠。

#### べき集合構成（Subset Construction）: NFA → DFA

Rabin-Scott（1959）の定理。NFAの状態集合の部分集合をDFAの1つの状態とみなす。

```
DFA状態 = P(NFA状態) の到達可能な部分
遷移: δ_DFA(S, a) = ε-closure(⋃_{q∈S} δ_NFA(q, a))
受理状態: NFA受理状態を含む部分集合
```

**最悪計算量**: n状態のNFAから2^n状態のDFAが生成される。この指数爆発は理論的に不可避 — 典型例は「末尾からn番目の文字が1」という言語。n+1状態のNFAで表現できるが、DFAには2^n状態が必要。

**実用上の緩和**: 到達可能な状態のみを生成する「遅延構成」（lazy construction）により、実際にはほとんどの場合で状態数は管理可能。RE2はさらにDFAの状態キャッシュにサイズ上限を設け、溢れたら再構成する戦略をとる。

#### 状態除去法: DFA → RE

一般化NFA（遷移ラベルに正規表現を許す）を経由する。非開始・非受理状態を1つずつ除去し、通過するパスを正規表現として遷移ラベルに吸収していく。最終的に開始状態→受理状態の遷移ラベルが全体の正規表現。

### ポンピング補題: 正規言語の限界

**定理**: 言語Lが正規ならば、ある定数p（ポンピング長）が存在し、p以上の長さの任意の文字列 s ∈ L に対して s = xyz と分割できて:
1. |xy| ≤ p
2. |y| > 0
3. 任意の i ≥ 0 に対して xy^i z ∈ L

**証明の本質**: DFAの状態数pに対し、長さp以上の入力はピジョンホール原理により必ずある状態を2回通る。そのループ部分がyであり、何回繰り返しても受理状態に到達する。

**用途**: ある言語が正規**でない**ことの証明に使う（背理法）。典型例: `{a^n b^n | n ≥ 0}` は正規でない（カウントが必要）。

**重要な注意**: 必要条件であって十分条件ではない。ポンピング補題を満たすが正規でない言語も存在する。十分条件にはMyhill-Nerode定理が必要。

### Chomsky階層: 計算能力の層構造

| Type | 文法 | オートマトン | 制約 | 例 |
|------|------|------------|------|-----|
| 3 | 正規文法 | 有限オートマトン（FA） | A → aB, A → a | `a*b+` |
| 2 | 文脈自由文法（CFG） | プッシュダウンオートマトン（PDA） | A → γ (任意の右辺) | `{a^n b^n}` |
| 1 | 文脈依存文法（CSG） | 線形有界オートマトン（LBA） | |α| ≤ |β| | `{a^n b^n c^n}` |
| 0 | 句構造文法 | チューリングマシン | 制約なし | 停止問題の補集合 |

**なぜ層構造が重要か**: 表現力と計算効率のトレードオフ。正規言語は O(n) で認識可能。CFLは O(n^3)（CYK）だが実用的にはLR(1)パーサで線形時間。CSLは PSPACE。型0は決定不能を含む。

**実用的含意**: コンパイラのフロントエンドが字句解析（正規言語）と構文解析（文脈自由言語）に分離されるのは、この階層の反映。正規言語で処理できるものをCFGで処理するのは無駄。

### 実用正規表現エンジン: 二つのパラダイム

#### Thompson NFAシミュレーション（オートマトン方式）

Thompson's constructionで生成したNFAを**直接シミュレート**する。各ステップでNFAの到達可能状態集合を追跡。

- **時間計算量**: O(mn) — mはパターン長、nは入力文字列長
- **空間計算量**: O(m) — 状態集合のサイズ
- **保証**: 入力に対して常に線形時間。病理的パターンが存在しない

Russ Coxの記事 "Regular Expression Matching Can Be Simple And Fast"（2007）がこの手法の再評価の契機となった。パターン `a?^n a^n` に対する `a^n` のマッチングで、backtrackingが O(2^n) かかるのに対しThompson NFAは O(n^2) で完了する有名なベンチマークを提示。

#### バックトラッキング方式（Perl/PCRE/Python/Ruby）

正規表現をプログラムとして解釈し、再帰的にバックトラッキングで探索する。

- **最悪計算量**: O(2^n) — 指数爆発
- **利点**: 後方参照（backreference）、先読み/後読み（lookahead/lookbehind）など形式的正規言語を超える機能を自然に実装できる
- **ReDOS（Regular Expression Denial of Service）**: 悪意あるパターンで指数時間を誘発する攻撃

**なぜバックトラッキングが主流になったか**: Larry Wallが Perl に組み込んだ際、後方参照という「便利な機能」を追加した。その後Python, Ruby, Java, JavaScript等が追従。Thompson NFAでは後方参照を処理できないため、利便性が計算量の保証に勝った歴史的経緯がある。

#### RE2: Thompson NFAの産業的実装

Russ Cox（Google）が開発。Thompson NFAに基づくオートマトン方式で、線形時間保証を提供。

**設計上の決断**:
- 後方参照を**サポートしない** — これは機能制限ではなく、線形時間保証を維持するための意図的設計
- DFAのオンデマンド構成（lazy DFA construction）: 状態をキャッシュし、メモリ上限に達したらキャッシュを破棄して再構成
- 固定スタックサイズ: マルチスレッドC++プログラムでのスタックオーバーフローを防止

**採用実績**: Go標準ライブラリ（regexp）、Google Code Search、Bigtable、PostgreSQL。信頼されない入力からの正規表現を安全に処理する必要がある場面で必須。

### 後方参照のNP完全性

後方参照 `\1`, `\2` 等は「以前にキャプチャした文字列と同一の文字列」のマッチを要求する。これは有限オートマトンの能力を超える — メモリが有限の機構では任意長の文字列を記憶できない。

Aho（1990）が正規表現 + 後方参照のマッチング問題がNP完全であることを証明。直感的には、後方参照で3-SATをエンコードできる。

**含意**: 後方参照を含む「正規表現」は、名前に反して正規言語を超えている。PCREの「正規表現」はChomsky階層のType 3ではなく、一部はType 1やそれ以上の表現力を持つ。

### 字句解析との接続

コンパイラの字句解析（lexer/scanner）は正規表現とオートマトンの最も重要な実用的応用:

1. **トークン定義**: 各トークン種別を正規表現で定義（`[0-9]+` → 整数リテラル、`[a-zA-Z_][a-zA-Z0-9_]*` → 識別子）
2. **NFA合成**: 全トークンの正規表現をThompson's constructionでNFAに変換し、和で合成
3. **DFA変換**: べき集合構成でDFAに変換（DFAは各入力文字に対して一意の遷移 → 分岐予測フレンドリー）
4. **DFA最小化**: Hopcroft（1971）のアルゴリズムで等価な状態を統合。O(n log n)
5. **テーブル駆動**: 遷移テーブルとしてコードに埋め込み

lex/flex はまさにこのパイプラインを自動化するツール。RE→NFA→DFA→最小DFA→遷移テーブルの全段階を実行し、Cソースコードを生成する。

**最長一致規則（maximal munch）**: 字句解析では最長マッチを採用。`>=` を `>` と `=` に分割しないために必要。DFAのシミュレーションで「最後に受理状態を通過した位置」を記録することで実現。

## 気づき・洞察

### 宣言と実行の分離
正規表現は「何を受理するか」の宣言的記述、オートマトンは「どう認識するか」の実行モデル。Kleeneの定理はこの二つが等価であることを保証する。SQLとクエリ実行計画、HTMLとレンダリングエンジンなど、宣言と実行の分離は計算機科学の普遍的パターンだが、その等価性が数学的に証明されている点が正規言語の特筆すべき性質。

### 表現力と決定可能性のトレードオフ
Chomsky階層は「何を表現できるか」と「その認識にどれだけのリソースが必要か」の根本的トレードオフを示す。正規言語は弱いが O(n) で認識でき、等価性・包含関係・空性がすべて決定可能。文脈自由言語では等価性が決定不能になる。表現力を得るたびに決定可能性を失う。

### 「正規表現」の名前の嘘
実用の「正規表現」（PCRE等）は形式的な正規言語を遥かに超えている。後方参照、先読み、原子グルーピング、条件分岐 — これらはType 3を超え、NP完全の領域にまで達する。名前の慣性が計算理論と実用の乖離を覆い隠している。RE2は「形式的正規言語に立ち戻る」ことで線形時間保証を取り戻した。

### 非決定性の計算論的意味
NFA→DFAの変換で状態数が指数爆発しうるという事実は、非決定性が本質的に計算をコンパクトに記述する能力を持つことを示す。P≠NP予想と同じ構造 — 非決定的に「正しい選択」を推測できるなら、探索空間を指数的に圧縮できる。

## 他分野との接続

- **B木・字句解析**: コンパイラのフロントエンドで正規表現→DFAの変換が使われるが、構文解析はCFG（Type 2）が必要。この階層分離がlex + yaccの二段構成を必然にしている。前回のB木エントリで触れたクエリ処理も、SQL解析は同じパイプラインを通る
- **TCP輻輳制御・ReDOS**: 正のフィードバックによるシステム崩壊パターン。バックトラッキング正規表現エンジンでの指数爆発は、悪意ある入力がシステムリソースを枯渇させる点でDoS攻撃と同構造。RE2が「信頼できない入力に対する安全性」を設計目標にしたのは、TCP輻輳制御がエンドホストの協調を前提にしたのと対照的
- **仮想メモリ・遅延構成**: RE2のlazy DFA constructionは、到達可能な状態のみオンデマンドで構成する。デマンドページングの「アクセスされるまでロードしない」と同じ遅延評価パターン。2^n個の理論上の状態を実際に使われる分だけ物質化する

## 次に深掘りしたいこと

- **Myhill-Nerode定理**: 正規言語の必要十分条件。ポンピング補題の弱点を補完する
- **正規言語の閉包性**: 和・連接・閉包・補集合・交差に対する閉包。文脈自由言語との閉包性の違い
- **Glushkov構成**: Thompson's constructionの代替。ε遷移のないNFAを直接生成
- **決定性プッシュダウンオートマトンとLR構文解析**: 文脈自由言語のうち決定的に認識可能な部分クラスとコンパイラの関係
- **ω正規言語とBüchiオートマトン**: 無限文字列に対する正規言語の拡張。モデル検査での応用
- **Hyperscan**: Intelの高性能正規表現エンジン。SIMD命令による並列マッチングの仕組み
