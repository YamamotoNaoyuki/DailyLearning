# FDTと摩擦係数：目標生地温度の計算と管理
**日付**: 2026-05-15  
**分野**: bread  
**タグ**: #DDT #FDT #摩擦係数 #温度管理 #ミキシング #発酵制御

## 学んだこと

### なぜ「最終生地温度（FDT）」が一つの数字に集約されるのか

プロのパン職人が日々の最重要管理項目として追跡する数字は、加水率でも発酵時間でもなく **DDT（Desired Dough Temperature, 目標生地温度）** と **FDT（Final Dough Temperature, 最終生地温度）** だ。これは「ミキシング直後の生地の中心温度」を指し、典型値は 24-26°C（リーンドゥ）、22-24°C（リッチドゥ）、24-25°C（サワードゥ）。

なぜこの数字が支配的か？それは**発酵速度が温度の指数関数**だから。Q10 ルール（生化学反応速度は温度10°C上昇で約2-3倍）が酵母とサワードゥ細菌の活動に近似的に成り立つ。生地温度が 22°C と 26°C の違いは、一見小さく見えて発酵時間にして約 40-60% の差を生む。**温度管理の精度こそが、一貫したパンを焼く最大の鍵**。

### DDT 計算式：「平均温度の制約条件」を解く

複数の温度源（小麦粉、水、室温、ルヴァン）が混合された生地の最終温度は、それらの加重平均に**ミキシング摩擦熱を加えた値**になる。これを逆算する公式が DDT 法：

#### 事前発酵（ルヴァン）なしの場合（3因子）
```
水温 = (DDT × 3) - (粉温 + 室温 + 摩擦係数)
```

#### 事前発酵あり（ルヴァン含む）の場合（4因子）
```
水温 = (DDT × 4) - (粉温 + 室温 + ルヴァン温度 + 摩擦係数)
```

#### 5因子（夏場の予冷小麦粉や追加要素）
DDT × N - (各温度ソースの合計) というように、要素数 N に応じて拡張される。

### 摩擦係数（Friction Factor）の正体

「摩擦係数（FF）」は係数というより**機械的ミキシングが生地に追加する温度量** を指す経験値。これはミキサーが生地に与える機械的エネルギーが熱に変換される量を表す。決定要因：

- **ミキサーの種類**: スパイラル < プラネタリー < 強力業務用
- **ミキサー速度**: 高速ほど発熱大（rpm に依存）
- **ミキシング時間**: 長いほど積算発熱大
- **生地量**: 多いほど摩擦面積大
- **加水率**: 低加水ほど摩擦大（粘性が高い）
- **生地のグルテン発達段階**: 強くなるほど摩擦増

### 摩擦係数の典型値（Hamelman 流）

ジェフリー・ハメルマン（『Bread』著者）の経験則：

- **業務用スパイラルミキサー（3分低速→4分中速）**: 約 24-28°F（≈ 13-16°C）の温度上昇
- **家庭用 KitchenAid 等プラネタリー**: 約 4-6°F（≈ 2-3°C）
- **手こね**: 0-4°F（≈ 0-2°C）
- **オートリーズ + ストレッチ&フォールド**: 1-3°F

注意：これは華氏での「上昇量」。摂氏に換算するときは差分なので 5/9 倍するだけ（オフセット +32 は不要）。

### 摩擦係数の実測法：自分のミキサーで測る

ハメルマンが推奨する手順：
1. 通常通り生地を仕込む
2. ミキシング直後に**中心温度を3回測定**して平均を取る（これが FDT）
3. 室温・粉温・水温・ルヴァン温度を記録
4. 計算式から逆引きする：

```
摩擦係数 = (FDT × N) - (粉温 + 室温 + 水温 + ルヴァン温度)
```

これを3-5回繰り返し、自分の機材・環境で平均値を出すと、それ以降は「水温だけ調整すれば DDT を狙える」状態になる。

### 実例計算

DDT = 25°C、粉温 = 21°C、室温 = 23°C、ルヴァン温度 = 24°C、摩擦係数 = 5°C（家庭用ミキサー、5分中速）

```
水温 = (25 × 4) - (21 + 23 + 24 + 5)
     = 100 - 73
     = 27°C
```

→ 水を27°C に加熱して投入すべし。

**夏場（室温30°C、粉温28°C）の同じレシピ**：
```
水温 = 100 - (28 + 30 + 28 + 5) = 100 - 91 = 9°C
```
→ 9°C の冷水（氷を加える必要あり）。

### 隠れた変数：氷の使用、容器の予冷、外気の影響

実際の運用では以下が摩擦係数からズレを生む：
- **氷を使う場合**: 氷の融解潜熱（80 cal/g）を考慮。氷 X% 添加すると追加冷却が大きい
- **生地の発酵で発生する熱**: ミキシング後の bulk fermentation 中も酵母代謝で温度が上昇する（典型 0.5-1.0°C/h）
- **ステンレスボウル vs プラスチック**: 熱伝導率が違うので外気との交換速度が異なる
- **オートリーズ時間**: 長く取ると生地が室温に近づくため、その後の水添加で再計算が要る

### Tartine 流の温度管理（実践哲学）

Chad Robertson の『Tartine Bread』では、家庭での精密温度計を持たない場合の**経験的指標**を勧める：
- 「手で触って体温よりわずかに冷たい」≈ 27°C
- 「室温よりやや暖かい」≈ 24-25°C
- フォルケンドルフ温度計（IR sensor or プローブ）を使えば ±0.5°C で測れる

プロは Cooper-Atkins などのプローブ温度計を必ず使い、ミキシング前・直後・bulk中・成形時に複数回測定する。

## 気づき・洞察

**「温度はパンの隠れたレシピ」**。多くのアマチュアレシピは「強力粉500g、水350g、塩10g、酵母3g、発酵2時間」と書くが、これは**室温・水温が前提通りである時のみ正しい**。同じレシピを夏冬で同じように作ると、発酵が完全に異なる挙動になる。プロのレシピは必ず「DDT 24°C」のような温度仕様を含む。**「時間で発酵を測る」のではなく「温度×時間で発酵を測る」** という発想転換が、再現性の鍵。

**摩擦係数は機材固有の「人格」**。同じ KitchenAid でも、年式・摩耗状態・ボウルサイズで摩擦係数は変わる。一度自分のミキサーで測定すれば、それは**自分の機材専用の校正値**になる。これは実験科学の「装置校正」と同じ作法。

**生地温度を中心とする一貫性の哲学**は、Service Mesh の「観測可能性（observability）」と同じ。「何が起きているか測定できる量を中心に据える」設計。パンで言えば、加水率・配合比は固定しても、温度を測定して逐次調整することで、毎日異なる外気条件で同じ品質を出す。

**DDT 計算式は熱力学的に不正確だが実用的に十分**。本来は比熱・質量・潜熱を考慮した熱量計算が必要だが、それは複雑すぎる。「全要素を単純平均してオフセット（摩擦係数）を足す」という近似が、職人技として通用している。これは工学一般の **「正確な物理よりも実用的なヒューリスティック」** の典型。

## 他分野との接続

- **anatomy / 体温調節**: 生地が酵母代謝で発熱するのと、人間が筋活動で発熱するのは同じ生化学的現象（基礎代謝率 BMR）。Q10 ルールはどちらにも成り立つ
- **golf / 環境調整**: 気温・湿度がボール飛距離に与える影響（夏冬で2クラブ違う）は、温度管理が必要な点で同じ。冷えた朝のラウンドではドライバーの飛距離が落ちる
- **music / 楽器の温度応答**: ピアノは室温で大きく音色変化（弦長変化、ハンマーフェルト硬度変化）。バッハの時代のクラヴィコードは温度に極めて敏感
- **tech / システム監視**: DDT は SLI/SLO の発想。「サービスの遅延 = DDT」「実測遅延 = FDT」「摩擦係数 = システム負荷」のアナロジー。観測 → 補正 → 期待値達成のループ
- **cs / フィードバック制御**: 摩擦係数の自己校正は、適応制御（adaptive control）の典型。システムの自己パラメータ推定
