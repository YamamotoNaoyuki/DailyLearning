# Google A2A（Agent-to-Agent）プロトコル
**日付**: 2026-03-05
**分野**: テクノロジー
**タグ**: #A2A #MCP #エージェント間通信 #プロトコル #相互運用性

## 学んだこと

### A2Aとは何か

Agent-to-Agent Protocol（A2A）は、2025年4月9日にGoogleが発表したオープンプロトコルで、異なるフレームワーク・異なるベンダーのAIエージェント同士が連携するための標準規格だ。発表時点で50以上のテクノロジーパートナー（Atlassian, Box, Cohere, Intuit, LangChain, MongoDB, PayPal, Salesforce, SAP, ServiceNow, Workday等）が支持を表明した。

### コアコンセプト

A2Aは4つの基本能力を提供する：

1. **Agent Card（エージェントカード）**: JSON形式のケイパビリティ記述。各エージェントが「何ができるか」を機械可読な形で公開する。OpenAPIのスキーマに相当
2. **Task Management（タスク管理）**: 定義されたライフサイクル状態を持つタスク管理。`submitted → working → input-required → completed/failed/canceled`
3. **Agent Collaboration（エージェント協調）**: クライアントエージェントがリモートエージェントにタスクを委任。コンテキストと指示を共有
4. **UX Negotiation（UX交渉）**: 異なるUI能力に適応。テキスト、フォーム、iframe等のアーティファクトを交換

### 技術スタック

既存の広く普及した標準の上に構築されている点が戦略的に重要：
- **HTTP**: トランスポート層
- **JSON-RPC**: メッセージフォーマット
- **SSE（Server-Sent Events）**: ストリーミング
- **v0.3（2025年7月）**: gRPCサポート追加、署名付きセキュリティカード

### A2A vs MCP——競合ではなく補完

これは最も重要な理解だ。MCPとA2Aは異なるレイヤーで動作する：

| 軸 | MCP | A2A |
|----|-----|-----|
| 方向 | 垂直（エージェント↔ツール） | 水平（エージェント↔エージェント） |
| 開発元 | Anthropic | Google |
| 目的 | 外部データソース・ツールへの接続 | エージェント間の相互運用 |
| 比喩 | ドライバー（USB-C） | 外交プロトコル |
| 透明性 | ツールの内部が見える | エージェントは不透明（opaque） |

**MCPは「手（ツール）」を繋ぐ。A2Aは「頭脳（エージェント）」を繋ぐ。**

あるエージェントがMCPでデータベースやAPIにアクセスし（垂直統合）、同時にA2Aで他のエージェントと協調する（水平統合）——これが想定されるアーキテクチャだ。

### ガバナンスと進化

- Apache 2.0ライセンス。Linux Foundation傘下
- v0.2: ステートレスインタラクション対応（セッション管理不要のシナリオ）
- v0.3（2025年7月）: gRPCサポート、署名付きセキュリティカード、拡張Python SDK。150+組織が参加
- A2AとAAIF（AI Agent Interoperability Framework、Linux Foundation）が競合的な立場に

## 気づき・洞察

A2AとMCPの補完関係は、前回学んだ信頼モデルの文脈で重要な意味を持つ。MCPは信頼されたツールとの接続（内部統合）、A2Aは信頼されていない可能性のある外部エージェントとの交渉（外交）だ。したがって、A2Aにはより厳格な認証・認可が必要になる——Agent CardはまさにDID（分散型識別子）と組み合わせるべきだろう。

「Agent Card」という概念は、人間社会の名刺に相当する。初対面で「私は何ができるか」を交換する。ただし機械可読であるため、人間の名刺よりもはるかに正確で検証可能だ。

## 他分野との接続

- **SDV**: 車載ECU間の通信は、まさにA2A的な「エージェント間プロトコル」。SOME/IPのService Discoveryは、A2AのAgent Cardに相当する機能を持つ。車載ドメインとAIエージェントドメインの設計パターンが収斂している
- **哲学**: A2Aの「opaque agent」——相手の内部実装を知らずに協調する——は、岡倉天心の「暗示する」文化に通じる。全てを開示せず、インターフェースだけで対話する
- **音楽**: オーケストラの各パートが独立した「エージェント」、指揮者が「オーケストレーター」、楽譜が「プロトコル」。A2Aはオーケストレーターなしの室内楽的協調を可能にする
