# 🌀 多重影分身・知識統合システム

## コンセプト
ナルトの多重影分身のように、各分野を独立して深く学びながら、その知識を一つに統合する。
各「影分身」は専門分野の情報を蓄積し、「本体」はその全てを統合した視点を持つ。

---

## ディレクトリ構造

```
kage-bunshin/
├── CLAUDE.md                    ← ここ（システム全体の指示）
├── domains/                     ← 各影分身（専門分野フォルダ）
│   ├── music/
│   │   ├── _summary.md          ← 分野サマリー（人間用の累積知識・常に最新を維持）
│   │   ├── _ledger.jsonl        ← カバレッジ台帳（機械可読・1エントリ1行・重複ゲートの照合先）
│   │   ├── _frontier.md         ← 未開拓フロンティアの待ち行列（テーマ選定の最優先）
│   │   └── YYYY-MM-DD_タイトル.md ← 日次エントリ
│   ├── piano/
│   ├── golf/
│   ├── anatomy/
│   ├── house-hunting/
│   ├── deutsch/
│   ├── other/
│   └── _archive/                ← 学習対象から外した過去ドメイン（tech / cs / bread / philosophy / sdv）
├── _integrations/               ← 統合レポート（本体の知恵）
│   └── YYYY-MM-DD_統合.md
└── _templates/                  ← テンプレート集
    ├── entry.md
    └── integration.md
```

## コマンド一覧（Claudeへの指示）

### 📥 学びを記録する
「〇〇について学んだことを記録して」と言うと：
1. **`domains/<分野>/_ledger.jsonl` と `_frontier.md` をロード**し、既習トピックと未開拓軸を把握
2. **重複ゲート**: テーマが台帳のキーワードと3語以上一致／同一クラスタなら「既習」。再訪は深掘り3条件（先行を `[[名指し]]`／新レイヤーを1つ／最深版が未存在）を満たす時のみ許可。**台帳にあるトピックを「未着手」と書かない**
3. 対象の `domains/<分野>/` に日付付きエントリファイルを作成（同日同ドメインに既存があれば停止＝冪等）
4. `_summary.md` を更新し、`python3 scripts/build_ledger.py <分野>` で `_ledger.jsonl` を更新、`_frontier.md` の待ち行列を消化・補充
5. 記録完了を報告

> ⚙️ 日次の自動学習（`scripts/daily_run.sh` → `/learn`）はこのフローを7ドメイン分まとめて実行する。詳細は `.claude/skills/learn/SKILL.md` の Step 1〜4 を参照。

### 📖 分野を振り返る
「〇〇分野のサマリーを見せて」と言うと：
- `domains/<分野>/_summary.md` を読んで要約

### 🔀 統合する（影分身を本体に戻す）
「全分野を統合して」または「影分身を戻して」と言うと：
1. 全分野の `_summary.md` を読み込む
2. 横断的なパターン・洞察・接続点を発見する
3. `_integrations/YYYY-MM-DD_統合.md` に保存

### 🔍 横断検索
「〇〇に関連することを全分野から探して」と言うと：
- 全ての `_summary.md` と最新エントリを横断検索
- 関連情報を分野ごとに整理して提示

### 📊 現状把握
「今の知識状況を教えて」と言うと：
- 各分野のエントリ数・最終更新日・サマリーを一覧表示

## Claudeへの重要な指示

1. **エントリ作成時**: 必ず `_summary.md` と `_ledger.jsonl`（`build_ledger.py` で）と `_frontier.md` も更新すること
2. **サマリーの形式**: 箇条書きで最新の知識を積み上げていく（過去のものも消さない）
3. **統合時**: 表面的なまとめではなく、**深い接続点と洞察**を重視する
4. **新分野追加時**: `domains/<新分野>/` フォルダと `_summary.md`・`_frontier.md` を作成し、`build_ledger.py` で `_ledger.jsonl` を初期化
5. **ファイル命名**: エントリは `YYYY-MM-DD_簡潔なタイトル.md` 形式
6. **ネタ被り防止（最重要）**: テーマ選定は必ず `_ledger.jsonl` と照合する。3語以上一致／同一クラスタは「既習」。**台帳にあるトピックを「未着手/分野初/空白」と書くことを禁止**。再訪は深掘り3条件（先行を `[[名指し]]` 参照・新レイヤー1つ以上・最深版が未存在）を満たす時のみ。同一クラスタは**8週間のクールダウン**。未開拓は `_frontier.md` の待ち行列を最優先で消化する
7. **台帳の運用**: `_ledger.jsonl` は機械可読の照合インデックス、`_summary.md` は人間用の知識資産。両者を混同しない。`scripts/audit_ledger.py` で既存重複を機械検出できる（週次フックで自動実行）
