# 🌀 多重影分身・知識統合システム

## コンセプト
ナルトの多重影分身のように、各分野を独立して深く学びながら、その知識を一つに統合する。
各「影分身」は専門分野の情報を蓄積し、「本体」はその全てを統合した視点を持つ。

---

## ディレクトリ構造

```
kage-bunshin/
├── CLAUDE.md                    ← ここ（システム全体の指示）
├── domains/                     ← 各影分身（専門分野フォルダ）
│   ├── tech/
│   │   ├── _summary.md          ← 分野サマリー（常に最新を維持）
│   │   └── YYYY-MM-DD_タイトル.md ← 日次エントリ
│   ├── music/
│   ├── piano/
│   ├── golf/
│   ├── anatomy/
│   ├── bread/
│   ├── philosophy/
│   ├── house-hunting/
│   ├── sdv/
│   └── other/
├── _integrations/               ← 統合レポート（本体の知恵）
│   └── YYYY-MM-DD_統合.md
└── _templates/                  ← テンプレート集
    ├── entry.md
    └── integration.md
```

## コマンド一覧（Claudeへの指示）

### 📥 学びを記録する
「〇〇について学んだことを記録して」と言うと：
1. 対象の `domains/<分野>/` に日付付きエントリファイルを作成
2. `domains/<分野>/_summary.md` を更新
3. 記録完了を報告

### 📖 分野を振り返る
「〇〇分野のサマリーを見せて」と言うと：
- `domains/<分野>/_summary.md` を読んで要約

### 🔀 統合する（影分身を本体に戻す）
「全分野を統合して」または「影分身を戻して」と言うと：
1. 全分野の `_summary.md` を読み込む
2. 横断的なパターン・洞察・接続点を発見する
3. `_integrations/YYYY-MM-DD_統合.md` に保存

### 🔍 横断検索
「〇〇に関連することを全分野から探して」と言うと：
- 全ての `_summary.md` と最新エントリを横断検索
- 関連情報を分野ごとに整理して提示

### 📊 現状把握
「今の知識状況を教えて」と言うと：
- 各分野のエントリ数・最終更新日・サマリーを一覧表示

## 統合レポートのフォーマット

```markdown

## 意外な接続点

全ての知識を合わせたときに見えてくるもの

## 次のアクション
各分野で深めるべきこと
```

## Claudeへの重要な指示

1. **エントリ作成時**: 必ず `_summary.md` も更新すること
2. **サマリーの形式**: 箇条書きで最新の知識を積み上げていく（過去のものも消さない）
3. **統合時**: 表面的なまとめではなく、**深い接続点と洞察**を重視する
4. **新分野追加時**: `domains/<新分野>/` フォルダと `_summary.md` を自動作成
5. **ファイル命名**: エントリは `YYYY-MM-DD_簡潔なタイトル.md` 形式
