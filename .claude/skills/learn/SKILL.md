# 🌀 影分身・日次学習スキル

全分野（tech, music, piano, golf, anatomy, bread, cs, house-hunting, deutsch, other）から**まだ学習していない新しいテーマ**を1つずつ選び、調査→記録→サマリー更新→レポート作成→Gist投稿までを一気に実行する。

## 追加指示
$ARGUMENTS

## 実行ステップ

### Step 1: 過去のエントリ確認
各 `domains/<分野>/` 配下の既存エントリファイル名を確認し、過去に扱ったテーマを把握する。
同じテーマ・類似テーマの重複を避けること。

### Step 2: 新テーマの選定
各分野から1つずつ新しいトピックを決める。テーマ選定の指針：
- **デフォルトは「遠いテーマ」**: 過去のエントリ群から**なるべく離れた新しい領域**のトピックを選ぶ。同じサブ領域に偏らないよう意識する（例: ピアノで和声の話が続いたら次は身体技法や練習法へ、ゴルフでショートゲームが続いたらルールやメンタルへ）
- `_summary.md` の「未解決の疑問」「次に深掘りしたいこと」は参考にするが、直近テーマの延長線上にあるものより、まだ手つかずの領域を優先する
- **ユーザーが追加指示（$ARGUMENTS）で「ドメインXは昨日のテーマと関連したものを」等と指定した場合のみ**、該当ドメインは前回テーマとの関連性を重視して選定する。指定のないドメインはデフォルトの「遠いテーマ」ルールに従う
- 具体的で掘り下げがいのあるトピックにする（広すぎるテーマは避ける）

#### `other` 分野の特別ルール（セレンディピティ枠）
`other` は他の分野に当てはまらない**完全にランダムな分野・トピック**を選ぶ。
- まず分野をランダムに決める（例: 天文学、料理化学、言語学、心理学、映画、建築、数学、生物学、経済学、歴史、地政学、写真、園芸、陶芸、香水、ワイン、将棋、格闘技、海洋学、火山学…何でもよい）
- 次にその分野の中から具体的で面白いトピックを1つ選ぶ
- 過去の `other` エントリと同じ分野を避け、毎回異なるジャンルから選ぶこと
- 目的は「新しい興味の種を見つけること」。既存分野との意外な接続が見つかれば最高
- もし特定のトピックが継続的に興味を引くようになったら、新しい専門分野として独立させることを提案する

### Step 3: 調査と記録
各分野について、WebSearch 等で最新情報を調査し、`domains/<分野>/YYYY-MM-DD_タイトル.md` を作成する。

#### ソース信頼性チェック
WebSearch の結果は**信頼性の高いソースのみ**から情報を採用すること。以下の基準で判定する：

**優先すべきソース（Tier 1）**:
- 学術論文・査読付きジャーナル（PubMed, Nature, Science, arXiv, Google Scholar 等）
- 公式ドキュメント・仕様書（RFC, W3C, 各プロジェクト公式サイト）
- 政府機関・公的機関のサイト（国交省、厚労省、FDA、WHO 等）
- 大学・研究機関の公開資料
- 業界の権威あるメディア（分野別: InfoQ/ACM/IEEE for tech, Grove Music Online for music, PGA.com for golf 等）

**許容するソース（Tier 2）**:
- 専門家が署名付きで書いた記事（著者の資格・所属が明記されている）
- 著名出版社の書籍・記事（O'Reilly, Cambridge UP, Springer 等）
- 評判の良い専門ブログ（長期運営、引用あり、著者が専門家）
- Wikipedia（概要の出発点としてのみ。詳細は引用元を追跡する）

**避けるべきソース（除外）**:
- 著者不明・匿名のブログ記事
- SEO目的のコンテンツファーム（薄い内容を大量生産しているサイト）
- AI生成が疑われる低品質な記事（内容が一般的すぎる、具体性がない）
- 明らかに古い情報（技術分野は2年以上前、法律分野は法改正前）
- 商品販売を主目的とするアフィリエイトサイト

**チェックプロセス**:
1. 検索結果の各ソースについて、ドメイン・著者・発行日を確認する
2. Tier 1 のソースを優先的に採用する
3. Tier 2 のソースは、Tier 1 と矛盾しない場合に補助的に使用する
4. 除外対象に該当するソースからの情報は使わない
5. 複数の独立したソースで裏付けが取れる情報を優先する（クロスチェック）
6. 各エントリの末尾に、主要な参考ソースを記載する

エントリはプロジェクトのテンプレート（`_templates/entry.md`）に従い、以下を必ず含む：
- 学んだこと（具体的で深い内容）
- 気づき・洞察（自分なりの解釈）
- 他分野との接続（横断的な視点）
- 次に深掘りしたいこと

### Step 4: サマリー更新
各 `domains/<分野>/_summary.md` を更新する。
- エントリ数を加算
- 最終更新日を今日の日付に
- 新しく学んだ知識を「蓄積された知識」に追加（過去のものは消さない）
- 「未解決の疑問」を更新

### Step 5: 統合レポート作成
`_integrations/YYYY-MM-DD_デイリーレポート.md` を作成する。
- 冒頭のトピック一覧テーブルに各エントリへの GitHub リンクを付ける
  - リンク形式: `https://github.com/YamamotoNaoyuki/DailyLearning/blob/main/domains/<分野>/ファイル名.md`
- 各セクション見出しにも `[[詳細]](リンク)` を付ける
- 全分野を横断する「今日の横断的パターン」セクションで深い洞察を提示する
- 末尾に `*Generated by 🌀 知識統合システム*` を付ける

### Step 6: Git コミット & プッシュ
新しいエントリ、更新したサマリー、レポートを git add → commit → push する。
コミットメッセージ例: `🌀 日次学習 YYYY-MM-DD: テーマ1, テーマ2, ...`

### Step 7: Gist 投稿
レポートを Gist に投稿する。
```
gh gist create "_integrations/YYYY-MM-DD_デイリーレポート.md" --desc "🌀 デイリーレポート YYYY-MM-DD" --public
```
投稿後、Gist の URL をユーザーに共有する。

## 注意事項
- 今日の日付は自動取得すること
- `music` 分野はクラシック音楽の鑑賞・分析・歴史に特化すること
- `piano` 分野はピアノ演奏の身体技法・奏法・練習法に特化すること（music分野との重複を避ける）
- `golf` 分野はコースマネジメント、クラブ選択、戦略、ルール、メンタルゲーム、プロツアーの分析など「ゴルフというゲーム」の理解に焦点を当てること。身体の使い方・バイオメカニクスは解剖学分野に統合済み
- `anatomy` 分野は日常動作・ピアノ演奏・ゴルフスイングなどの身体最適化を解剖学的に扱うこと。golf/piano分野と連携する横断的な視点を重視する
- `cs` 分野はコンピュータサイエンスの基礎を体系的に学ぶ。ユーザーは工学部大学院卒の基礎があるため、初歩的すぎる説明は不要。以下の領域を幅広くカバーする：
  - アルゴリズムとデータ構造（計算量解析、グラフ理論、動的計画法、etc.）
  - オペレーティングシステム（プロセス管理、メモリ管理、ファイルシステム、etc.）
  - コンピュータネットワーク（TCP/IP、ルーティング、DNS、TLS、etc.）
  - データベース（関係モデル、トランザクション、インデックス、分散DB、etc.）
  - コンパイラ・言語処理系（字句解析、構文解析、型システム、最適化、etc.）
  - 分散システム（合意アルゴリズム、CAP定理、一貫性モデル、etc.）
  - 計算理論（オートマトン、チューリングマシン、計算可能性、NP完全性、etc.）
  - コンピュータアーキテクチャ（キャッシュ、パイプライン、分岐予測、SIMD、etc.）
  - セキュリティ（暗号理論、認証、脆弱性モデル、etc.）
  - 各トピックは「なぜそう設計されたか」という動機・トレードオフを重視し、教科書的な定義の列挙ではなく、設計判断の背景と実践的な意味を深掘りすること
  - tech 分野（AI・最新技術トレンド）との差別化：cs は「時代を超えた基礎理論・原理」に焦点を当てる
- `deutsch` 分野はtelc B1取得済みのドイツ語学習者向けの毎日フレーズ学習に特化すること。毎回異なるテーマ（雑談・職場・意見・感情・不満・予定・食事・ニュース・買い物・旅行・健康・趣味・天気・お金・引っ越し・近所付き合い等）を選び、「教科書的でない、ネイティブが実際に使う」フレーズを4つ生成する。各フレーズには以下を含めること：
  - 日本語訳
  - 単語リスト（主要単語の意味）
  - 使う場面（具体的なシチュエーション）
  - フォーマル度（formal / neutral / umgangssprachlich / slang の4段階）
  - キーワード解説（格変化・前置詞の注記あり、文法ポイント）
  - 関連表現（同じ意味の別の言い方、バリエーション）
- `deutsch` 分野のエントリ形式は他分野と異なる。テンプレートの「学んだこと」セクションにフレーズ4つを記載し、「気づき・洞察」に文法的な気づきやドイツ語特有の表現パターン、「他分野との接続」にドイツ文化・哲学・音楽等との接点を記載する
- `deutsch` 分野のサマリーには、蓄積されたフレーズ数、カバー済みテーマ、重要文法パターンを記録する
- 各エントリは読み応えのある深さにすること（箇条書きの羅列ではなく、ストーリーのある内容）
- 他分野との接続は表面的なこじつけではなく、本質的な共通点を見出すこと
